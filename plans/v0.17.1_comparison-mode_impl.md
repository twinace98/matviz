## 17.3.0 — NN atom matching 알고리즘

**파일:**
- `src/webview/nnMatching.ts` (신규, ~80 LOC) — `matchByNN()` + spatial bin
  helper.
- `scripts/test-nn-matching.ts` — 4 unit test (identity, drift, swap, large-N).
- `esbuild.mjs` — `nnMatchingTestConfig` 엔트리.

**API:**
```ts
export interface MatchResult {
  pairs: Array<{ a: number; b: number; displacement: [number, number, number] }>;
  // a is primary atom index, b is secondary atom index, displacement = posB - posA.
  unmatched: number[];  // primary indices that found no match within threshold.
}

export function matchByNN(
  primarySpecies: string[],
  primaryPos: [number, number, number][],
  secondarySpecies: string[],
  secondaryPos: [number, number, number][],
  threshold: number = 2.0,    // Å
): MatchResult;
```

**알고리즘:**
1. Group atoms by species in both primary and secondary.
2. For each species in primary: greedy NN matching against secondary's
   same-species atoms.
3. NN search: naive O(N²) for N≤100, spatial bin O(N) above.
4. `usedSecondary: Set<number>` 로 매칭된 secondary 추적 (greedy + 결정성).
5. NN distance > threshold → unmatched에 primary 인덱스 추가.

**Spatial bin (N>100):**
- Bin size = max(threshold, 2.0) Å (검색 반경에 맞춤).
- For each primary atom: 자신의 bin + 26 인접 bin만 검색.

**통과 기준 (test-nn-matching.ts 4 케이스):**
- **Identity**: same atoms in same order → all pairs i↔i, displacement [0,0,0],
  unmatched=[].
- **Uniform drift**: secondary positions = primary + (0.5, 0, 0) → all pairs
  i↔i, displacement [0.5, 0, 0].
- **Swap**: 두 같은-species atom의 위치가 secondary에서 swap → pairs는 위치
  기반 정확 매칭 (i ↔ swapped index).
- **Large-N spatial bin**: 200 random atoms, primary와 secondary가 atom
  순서만 셔플된 동일 위치 → 결과 displacement 모두 [0,0,0] within tol.

## 17.3.1 — Displacement arrow renderer + UI 통합 (closes v0.17.1)

**파일:**
- `src/webview/displacementArrowRenderer.ts` (신규, ~80 LOC) —
  magneticArrowRenderer 구조 재사용 (cone tip + cylinder shaft InstancedMesh).
  주요 차이: 색상은 Viridis-only (sign 없음), magnitude scale 1.0 Å/Å.
- `src/webview/renderer.ts` — comparison state (current MatchResult + onset
  보존), `setComparisonActive(b)` API, frame change hook (setFrame이 active
  comparison 재계산).
- `src/webview/message.ts` — `compareToPhase` + `clearComparison` 메시지.
- `src/webview/main.ts` — 메시지 dispatch.
- `src/extension.ts` — `matviz.compareToPhase` + `matviz.clearComparison`
  명령. compareToPhase는 secondary phase가 있는지 가정 (사용자가 먼저
  addPhase 호출 권장 — 없으면 토스트 안내).
- `package.json` — 두 명령 등록.

**렌더 dispatch:**
- renderer: `comparisonGroup: THREE.Group` 추가, scene.add 등록.
- `compareToPhase()` API: 첫 secondary phase가 있으면 NN matching 호출 →
  MatchResult 보존 → `displacementArrowRenderer.rebuild(pairs)` 호출.
  매핑 없으면 console.warn + return.
- `setFrame()` 끝에 (현재 active comparison 있으면) `recomputeComparison()`
  호출 → 같은 secondary 대비 새 frame primary 위치로 재매칭.

**UI:**
- 명령 팔레트 진입 (사이드 패널 UI는 v0.17.x).
- compareToPhase: secondary 있으면 진입 + 자동 활성화. 없으면 "Add a phase
  first via 'MatViz: Add Phase'" 토스트.
- clearComparison: comparison group + 상태 클리어.

**Frame-aware 동작:**
- Primary가 trajectory + active comparison 인 경우, 매 setFrame 후 NN
  재계산 (secondary는 항상 frame 0 또는 single-frame phase).
- atom count 변화 시 (NPT trajectory에서는 atom count 안 바뀜) 안전.
- Auto-disable: `getAtomCount() > 5000`이면 compareToPhase 명령이 토스트
  + return.

**통과 기준:**
- `test/fixtures/test-trajectory.axsf` 열기 → addPhase로 nacl.cif 추가 →
  compareToPhase 명령 → 변위 화살표 표시.
- AXSF frame slider 드래그 → 화살표 자동 재계산 (frame 0 = no displacement,
  frame 3 = max displacement).
- clearComparison → 화살표 사라짐.
- 시각 회귀 (8 fixtures): 0 (모든 신규 코드는 명령 진입식 — 자동 트리거 없음).
