## Feature 6 — Panel content surfaces (style chips, iOS switches, V2 supercell, info pill, gizmo)

This bundles five panel-content changes into one commit since they're all
restyles of items that already exist and share the panel restructure.

### 6a — Style chips
Replace `<select id="display-style">` with:
```html
<div class="chips" id="display-style-chips">
  <button class="chip active" data-style="ball-and-stick">Ball &amp; Stick</button>
  <button class="chip" data-style="space-filling">Space-filling</button>
  <button class="chip" data-style="stick">Stick</button>
  <button class="chip" data-style="wireframe">Wireframe</button>
</div>
```

CSS lift from `v2.css:257–273`. JS rewrite in `main.ts`: replace
`styleSelect.addEventListener('change', ...)` with delegated click handler
that toggles `.active` and dispatches the same renderer call. Saved-state
read/write keys unchanged.

### 6b — iOS toggle switches
For every `.toggle input[type=checkbox]` in the side panel (Bonds, Labels,
Polyhedra, Boundary, Cell lines, Ellipsoids, Partial occupancy, Loop,
Recompute bonds, Compare, Magmom, Impostor, plus per-element atoms list):

- Hide the native checkbox: `.toggle input { position: absolute; opacity: 0;
  pointer-events: none }`.
- Render a `<span class="switch"><span class="switch-thumb"/></span>` after
  the checkbox; toggle `.switch.on` via JS in response to `change`.
- Style with V2 dimensions: 36×20 default, 30×18 `.sm` for atoms list.
  Color: `--vscode-focusBorder` when on (`v2.css:286–314`).

Implementation strategy: a one-time `setupToggleSwitches()` pass after DOM
ready that walks all `.toggle input[type=checkbox]`, inserts the switch span,
and wires the visual sync. No per-call-site changes needed.

### 6c — V2 supercell stepper (− N +, no upper bound)
Replace the V1 `.num-wrap` for sc-a/b/c (introduced in Feature 1) with V2's
horizontal stepper:
```html
<div class="sc-row">
  <div class="sc">
    <div class="sc-l">a</div>
    <div class="sc-steppers">
      <button class="sc-dec" data-axis="0">−</button>
      <input type="text" inputmode="numeric" class="sc-val" value="1" id="sc-a">
      <button class="sc-inc" data-axis="0">+</button>
    </div>
  </div>
  <!-- b, c -->
</div>
```

`.sc-val` is now an `<input>` (kept editable per existing JS contract: code
reads `.value`, can write back via stepper buttons). Width clamped via
`width: 100%; text-align: center; font-variant-numeric: tabular-nums`.

**No upper bound:** `min: 1, max: Infinity` (or simply omit max). Memory
cost: each axis bump multiplies render geometry — current 5-cell cap was
defensive. Document the removal in the commit message and `STATUS.md`
known-issue line so a future regression report has context. (Spatial-hash
bond detection still hard-skips at 5000 atoms, which is the real safety
net.)

### 6d — Bottom-left info pill (canonical formula readout)
**Decision (locked 2026-04-26):** the pill is the **only** place the formula,
space group, atom count, and volume are displayed. The side panel no longer
has a `panel-head` block (per Feature 5). The pill is **always visible**
regardless of whether the side panel is open or closed.

New element after `#side-panel`:
```html
<div id="info-pill">
  <span class="pill-formula" id="pill-formula"></span>
  <span class="pill-meta" id="pill-meta"></span>
</div>
```

CSS lift `v2.css:386–416`. Position `bottom: 16px; left: 52px`; when the
side panel is open, shift to `left: 316px` so the pill sits to the right of
the panel rather than under it. Implement via a body class `.panel-open`
(toggled by `#panel-toggle` JS) since `~` selectors don't cross unrelated
DOM ancestors.

JS: search `main.ts` for `getElementById('info')` and redirect the formula
write to `#pill-formula` (formula text) + `#pill-meta` (` · `-separated spg,
atom count, volume).

### 6e — Axis gizmo bottom-right
Currently `axisIndicator.ts` mounts at bottom-left (per its own constructor
or wherever `main.ts` parents it). Move parent container to a new
`<div id="axis-gizmo-host">` with `position: absolute; bottom: 16px; right:
16px; padding: 8px; background: var(--v2-glass-1); backdrop-filter: blur(
var(--v2-blur-sm)); border: 1px solid var(--v2-line); border-radius:
var(--v2-r-sm)`.

Check `axisIndicator.ts` for hard-coded position assumptions — V2 wraps it
in a glass card; the indicator itself should be position-agnostic.

## Feature 8 — Measure HUD overlay

**Files:**
- `crystalEditorProvider.ts` — add `<div id="measure-hud" class="hidden">…</div>`
- `media/styles.css` — `.measure-hud` rules (lift `v2.css:437–636` verbatim,
  rename `.v2m*` → `.measure-hud-*`)
- `src/webview/main.ts` or new `src/webview/measureHud.ts` — wire selection
  state to HUD content
- `src/webview/picking.ts` — emit a typed `selectionChanged` signal HUD
  subscribes to (avoid polling)

### HUD content (V2 spec, `V2_Floating.jsx:233–299`)
- **Head**: kind switcher (Distance / Angle / Dihedral) — Distance always
  enabled when ≥2 atoms; Angle enabled iff exactly 3 selected; Dihedral iff
  4. Active kind highlighted with `--v2-surf-accent` + `--v2-line-focus`
  border. ✕ button clears selection.
- **Hero value**: 34px gradient-text number + 20px Å unit (or ° / dihedral
  unit), `font-variant-numeric: tabular-nums`.
- **Atom pair card** (only for distance/angle/dihedral with ≤2 visible
  rows): swatch + symbol/index pairs joined by a connector with ↔ glyph.
- **Δ readouts**: fractional and cartesian rows when distance is shown.
  Computed via existing measurement utilities — verify the math is already
  available in `picking.ts` or a measurement helper before adding new
  computation.
- **Footer**: hint text ("Click a 3rd atom for **angle** · 4th for
  **dihedral**") + Copy button.

### Wiring
- Picking layer keeps the source of truth for selected atoms.
- HUD subscribes via either:
  - **(A) DOM event** `picking.ts` dispatches on a known node (e.g.
    `canvas.dispatchEvent(new CustomEvent('mv:selection', { detail: { … } }))`);
    HUD listens. Loosely coupled; no import cycles.
  - **(B) Direct callback** `picking.ts` accepts an `onSelectionChange`
    handler at construction; HUD passes one in.
  - Decision in this version: **(A)** — fewer lifetime concerns, keeps HUD
    optional.
- Existing yellow/cyan/magenta dashed measurement overlays (v0.15.0) stay
  unchanged — they're 3D scene artefacts; HUD is the 2D readout.

### Copy button
`navigator.clipboard.writeText(...)` of a tab-separated readout. Webview CSP
allows clipboard write under user-gesture; verify before claiming done.

### Visibility
HUD shown iff `mode === 'measure' && selectedAtomCount >= 2`. Hidden via
existing `.hidden { display: none !important }` utility.
