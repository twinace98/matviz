## 17.4.1 — `--phase <file>` 다중 phase overlay

**파일:**
- `scripts/render.ts`:
  - `RenderOptions.phases: string[]` (default `[]`).
  - parseArgs `--phase <file>` 반복 — args[i] === '--phase'면 push.
  - Help text 업데이트.
  - Inline HTML OPTS JSON에 `phases: Array<{species, positions, lattice, ...}>`
    추가 (file 읽기 + parseStructureFile + 첫 frame만 + offset/opacity hardcode
    `[0,0,0]` / `0.5`).
  - Inline render: secondary phase atoms 렌더 (transparent Phong), v0.17.2
    webview의 secondaryPhases rebuild 로직을 inline 재현 (~40 LOC).
- `skills/matviz-render/SKILL.md` + `README.md` — `--phase` 플래그 + 예제.
- `test/visual/fixtures/cli-phase-overlay/scene.json` — primary nacl.cif +
  `phase: ["../../../fixtures/test-occupancy.cif"]`.
- `test/visual/baseline/cli-phase-overlay.png`.
- `test/visual/harness.ts` — `phases?: string[]` SceneConfig 필드 + buildArgs
  매핑.

**Inline render approach:**
v0.17.2 webview의 `rebuildSecondaryPhases()` 패턴을 CLI에 inline 포팅:
- Per-element InstancedMesh of transparent Phong sphere.
- Position = phase atom + offset (default 0).
- Opacity = phase opacity (default 0.5).
- depthWrite: false, renderOrder: 1.

**통과 기준:**
- 단일 `--phase` 파일 → primary 위에 secondary 반투명 표시.
- 다중 `--phase a.cif --phase b.cif` → 두 secondary 모두 표시.
- 비-trajectory primary와 동일 fixture 회귀 0.

**작업 단위 추정:** 1d.

## 17.4.2 — `--compare-to-phase` 변위 화살표 + stdout RMSD summary

**파일:**
- `scripts/render.ts`:
  - `RenderOptions.compareToPhase: boolean` (default false).
  - parseArgs `--compare-to-phase`.
  - Pre-flight: `compareToPhase && phases.length === 0` → error + exit 1.
  - `compareToPhase && allFrames` → error + exit 1.
  - Inline HTML OPTS JSON에 `compareToPhase: true` + 첫 phase 데이터.
  - Inline NN matching + displacement arrow render (~80 LOC, displacement
    Viridis colormap, 17.2.2 PBC inline).
  - Stdout RMSD summary 출력 (page.evaluate로 stats 객체 read → console.log).
- `skills/matviz-render/SKILL.md` + `README.md` — `--compare-to-phase` 예제 +
  RMSD stdout 활용 (예: bash script에서 grep + parsing).
- `test/visual/fixtures/cli-comparison/scene.json` — primary nacl.cif +
  `phase: ["../../../fixtures/nacl.cif"]` (자기-자기) + `compareToPhase: true`.
- `test/visual/baseline/cli-comparison.png` — 자기-자기 비교는 displacement 0
  → 화살표 없음 → primary + secondary 동일 위치 overlay만 보임.

**Inline NN matching:**
v0.17.1.0의 nnMatching.ts와 17.2.2의 PBC 로직을 CLI inline (Three.js
이미 inline import 됨, nnMatching pure JS이라 inline 가능).

**Stdout summary format:**
```
[comparison] RMSD: 0.000 Å (matched 8, unmatched 0, max 0.000 Å, mean 0.000 Å, p95 0.000 Å)
```
한 줄, machine-parseable + human-readable.

**통과 기준:**
- 자기-자기 비교 → displacement ~0, RMSD 0, 화살표 없음.
- relaxed vs unrelaxed (서로 다른 lattice) → 변위 화살표 + non-zero RMSD.
- `--compare-to-phase` without `--phase` → exit 1 + clear error.
- `--all-frames --compare-to-phase` → exit 1 + clear error.

**작업 단위 추정:** 1d.
