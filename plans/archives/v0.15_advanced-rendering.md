# v0.15 — 고급 렌더링

**목표:** 10 k+ 원자 구조에서도 체감 가능한 속도로 렌더·상호작용. VESTA가 버벅거리기 시작하는 규모에서 우리 뷰어가 매끄럽게 굴러가는 것이 지표.

**상태:** 킥오프 초안 — 착수 전 승인 대기.

## 범위

`Plan.md` v0.15 표 + 기술부채 레지스트리 재배치:

| # | 피처 | 통과 기준 |
|---|------|-----------|
| 15.0 | 준비 리팩터링 — `BondRenderer` 추출 (`MaterialRegistry` 포함) | `renderer.ts` 200+ LOC 감소; 시각적 회귀 없음 |
| 15.1 | Sphere impostor (billboard + fragment shader) | 10 k 원자 삼각형 수 10× 이상 감소; 외곽 픽셀 일치; 양쪽 팔레트 OK |
| 15.2 | GPU 피킹 (render-to-texture로 instance id 읽기) | 50 k 원자에서 클릭 → 응답 < 5 ms |

| 15.5 | Cylinder impostor (bond) | 50 k bond 에서 triangles 10× 감소; bicolor/unicolor 시각 동일; line 모드 유지 |

## 피처 간 의존성

- **15.0 먼저** — BondRenderer를 먼저 떼면 15.1 impostor 도입 시 bond 경로가 분리되어 변경 폭 작아짐.
- **15.1 → 15.3** — impostor는 frustum 비용도 낮추므로 같이 가기 좋음. 15.3은 15.1 직후.
- **15.2 독립** — 현재 raycaster와 공존 가능 (N < 5000 fallback).
- **15.4 마지막** — 결정 게이트. WebGL2 파이프라인이 안정된 뒤에 평가.

**제안 순서:** 15.0 → 15.1 → 15.3 → 15.2 → 15.4.

## 결정 게이트

1. **15.4 종료 시점** — WebGPU로 모든 test fixture 정상 렌더 && 성능 ≥ WebGL2 경로? 
   - 통과: WebGPU를 feature flag 뒤에 동봉 (`matviz.experimental.webgpu`). 이후 버전에서 기본 전환 고려.
   - 실패: WebGPU 코드 제거, 6개월 후 재검토.
2. **15.1 롤아웃 방식** (소규모 내부 게이트) — impostor를 기본값 vs 고원자수 자동 스위치. 초기 제안: **N ≥ 2000 일 때 자동 impostor**, 사용자 토글로 강제 on/off 가능.

## 착수 전 확정 필요 질문

1. **15.0 리팩터링 범위** — BondRenderer만 떼고 MaterialRegistry는 유지? 아니면 둘 다 신규 모듈로? **제안: BondRenderer만 우선 추출 (cylinder / line / wireframe 경로 일원화), MaterialRegistry는 v0.15 말미 여유 있으면 추가.**
2. **15.1 impostor 기본 활성 조건** — N ≥ 2000 자동 / 항상 / 항상 끔? **제안: N ≥ 2000 자동 + 사용자 토글 (사이드 패널 Style 섹션)**.
3. **15.2 raycaster 대체 여부** — GPU 피킹이 성공하면 raycaster 제거? **제안: 유지** — N < 5000일 때 raycaster (단일 호출 비용이 더 작음), 그 이상은 GPU 피킹. 코드 복잡도 소량 증가는 수용.
4. **15.4 범위** — 순수 프로토타입 (branch에서만) vs 기능 플래그로 동봉? **제안: 기능 플래그로 동봉**, 사용자가 `matviz.experimental.webgpu=true` 때만 활성.
5. **성능 타깃 재설정** — `Plan.md` 기존 목표 "50 k 원자 회전 중 30 fps". v0.14 스트레스 픽스처 16 k 원자 SwiftShader 수치를 베이스라인으로 먼저 측정 후 확정? **제안: 15.0 완료 직후 16 k 원자에서 베이스라인 측정 → 목표는 "50 k 원자 회전 중 SwiftShader 15 fps / 실 GPU 30 fps"로 완화**.

구체 파일/코드 레시피는 `v0.15_advanced-rendering_impl.md` 참조.
