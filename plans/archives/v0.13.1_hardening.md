# v0.13.1 — Hardening Hotfix

## Goal

2026-04-17 전면 코드 리뷰에서 확인된 **정확성·신뢰성 결함**을 v0.14 UX 폴리시 시작 전에 먼저 닫는다. UX 변경은 일절 포함하지 않는 작고 경계가 분명한 패치 릴리스.

## Scope

### 포함

1. **원소 테이블 단일화 (Single source of truth)**
   - 현재 3중 중복: `src/parsers/elements.ts` (Node 80개) / `src/webview/elements-data.ts` (browser 68개, lanthanides 12개 누락) / `scripts/render.ts` 내부 테이블 (CLI 57개 + dr drift).
   - webview에 Pr, Nd, Sm, Eu, Gd, Tb, Dy, Ho, Er, Tm, Yb, Lu 누락 → 희토류 포함 구조는 placeholder 색으로 렌더됨.
   - CLAUDE.md 동기화 규칙을 코드로 강제.

2. **Editor 에러 바운더리**
   - `crystalEditorProvider.ts:52`의 `parseStructureFile()` 호출이 try/catch 없음.
   - 파서 throw 시 에디터 탭이 사용 불가 상태가 되며, 사용자에게 메시지가 표시되지 않음.
   - `*.out` 연관은 모든 `.out` 파일을 QE로 넘기므로 실제 발생 빈도가 높음.

3. **Ambiguous 확장자는 텍스트 기본 + "Open in MatViz" 버튼**
   - `*.out`, `*.in`, `*.stdin`, `*.stdout`, `geometry.in`은 `customEditors.priority: "option"`으로 변경 — 텍스트 에디터가 기본.
   - Editor title-bar에 **"Open in MatViz"** 버튼을 추가하여 한 번 클릭으로 MatViz 전환.
   - 명확한 포맷(`*.cif`, `*.xsf`, `*.poscar`, `POSCAR`, `CONTCAR`, `*.xyz`, `*.pdb`, `*.cube`, `*.cub`, `CHGCAR`, `AECCAR0`, `AECCAR2`, `PARCHG`)은 계속 `priority: "default"`로 유지 — 기존 사용자 워크플로우 회귀 없음.
   - 버튼 표시 조건은 단순 확장자 매칭 (content signature 검사는 부하·활성화 범위 문제로 배제). 사용자가 버튼을 눌렀는데 파싱이 실패하면 13.1.2 에러 바운더리가 처리.

4. **Renderer inline material registry**
   - `renderer.ts` 전반에 inline으로 생성된 Material 11종이 `this.materialCache`에도 `this.geometries`에도 등록되지 않아 `disposeResources()` 시 누락됨.
   - `rebuild()`는 supercell/iso-level/style/palette 변경마다 호출되므로 상호작용 중 VRAM leak.
   - 위치: `renderer.ts:425, 499, 517, 864, 873, 890, 899, 1152, 1229, 1676, 1763, 1775, 1785, 1849, 1897`.

5. **CLI renderer 브라우저 생명주기**
   - `scripts/render.ts:717-792`에서 `browser.close()`가 happy path에만 존재.
   - `fs.readFileSync` / `parseStructureFile` / `page.goto` / `waitForFunction` 중 throw 시 Chromium이 outer `process.exit(1)`에 의존해야만 종료됨 (라이브러리화 시 문제).

6. **CSP 강화 (style)**
   - `crystalEditorProvider.ts:130`의 `style-src ${webview.cspSource} 'unsafe-inline'`에서 `'unsafe-inline'` 제거.
   - styles.css는 외부 파일이므로 inline style이 실제로 필요하지 않음.

7. **QE 파서 silent-success 제거 (부가)**
   - `qeParser.ts:85-86`이 파싱 실패 시 10×10×10 default lattice를 반환함.
   - throw로 변경해야 (2)와 (3)의 에러 바운더리가 정상 동작.

## 기술적 결정

### 원소 데이터 단일화 방식

두 가지 선택지:

| 방안 | 장점 | 단점 |
|------|------|------|
| A. `src/shared/elements-data.ts`를 만들고 Node/browser/CLI 세 번들이 모두 import | 단순, esbuild가 platform별로 알아서 처리 | 기존 `elements.ts`의 Node-only 유틸 함수 재배치 필요 |
| B. build step에서 한 JSON 파일로부터 세 tsc 산출물을 생성 | 타입 분리 명확 | 빌드 파이프라인 복잡도 증가, 디버깅 어려움 |

**권장: A**. `CrystalStructure` 타입이 이미 Node/browser 양쪽에서 import되고 있어 공유 모듈 패턴 선례 존재. CLI 번들은 현재 inline 테이블을 완전히 제거하고 shared 모듈을 import.

### Ambiguous 확장자 처리

`package.json` `contributes.customEditors`를 두 개의 selector 블록으로 분리:
- **Default 블록** (`priority: "default"`): 현재와 동일한 명확 포맷만.
- **Option 블록** (`priority: "option"`): `*.out`, `*.in`, `*.stdin`, `*.stdout`, `geometry.in`.

`contributes.menus.editor/title`에 `matviz.openInViewer` 명령어 버튼 추가. `when` 절은 `resourceExtname` 매칭으로 해당 확장자에서만 표시. 명령 핸들러는 `vscode.commands.executeCommand('vscode.openWith', uri, 'matviz.crystalViewer')` 한 줄.

**`src/parsers/index.ts`의 기존 `.out`/`.in` 분기는 유지** — 버튼 클릭 → customEditor → `openCustomDocument` → `parseStructureFile` 경로에서 여전히 호출되기 때문. 단, QE 시그니처 체크는 여기서도 도입하여 더 명확한 에러 메시지를 제공 (아래 13.1.4 참조).

### Material registry 설계

최소 침습: `renderer.ts`에 `private disposables: Array<THREE.Material>` 추가.
- Inline material 생성 직후 `this.disposables.push(mat)`.
- `disposeResources()`에서 `for (const m of this.disposables) m.dispose(); this.disposables = [];`.
- 클래스 추출은 v0.14.7 AxisIndicator 작업과 동시에.

## Exit criterion

1. 모든 7개 sub-step이 수동 테스트 통과.
2. `test/fixtures/`에 희토류 포함 CIF 1개 추가 (예: CeO2 또는 Nd2Fe14B) 후 webview + CLI 모두 정상 렌더.
3. `npm run build && npx tsc --noEmit` 둘 다 0 error.
4. 기존 fixture(nacl, silicon, graphene, buckyball, alanine, h2o, LiF polaron) 회귀 없음.
5. `vscode-matviz-0.13.1.vsix` 패키지 후 수동 설치 → 모든 기존 기능 동작.
