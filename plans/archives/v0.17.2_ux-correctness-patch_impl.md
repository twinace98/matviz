## 17.2.2 — PBC-aware NN matching (minimum-image distance)

**파일:**
- `src/webview/nnMatching.ts`:
  - `matchByNN`에 `lattice?: [number, number, number][]` 옵션 추가.
  - lattice 있으면 displacement 계산 시 minimum-image:
    1. cartToFrac(d, lattice)
    2. d_frac → d_frac - round(d_frac) (per axis)
    3. fracToCart(d_frac', lattice)
  - lattice 없으면 기존 raw cartesian (현재 동작).
- `src/webview/renderer.ts`:
  - `recomputeComparison`이 primary와 secondary가 같은 lattice ref + 같은
    cell volume이면 lattice를 matchByNN에 전달.
- `scripts/test-nn-matching.ts` — PBC wrap 테스트 추가.

**Cell-mismatch 가드:** Primary와 secondary가 다른 cell이면 PBC 적용 무의미
→ raw cartesian. fixed-cell trajectory + 같은 결정 family 비교에서만 PBC
혜택.

**통과 기준:**
- NPT XDATCAR fixture에서 atom이 cell wrap한 경우 변위가 작은 값으로
  계산 (PBC 없을 때 inflate 방지). 단위 테스트 1개 추가.

**작업 단위 추정:** 0.5d.

## 17.2.3 — RMSD/displacement summary panel

**파일:**
- `src/webview/renderer.ts`:
  - `recomputeComparison` 끝에 `lastComparisonStats` 계산 + 저장.
    ```ts
    interface ComparisonStats {
      rmsd: number;            // Å, sqrt(mean(d²)) over matched pairs
      maxDisplacement: number;
      meanDisplacement: number;
      p95Displacement: number;
      matchedCount: number;
      unmatchedCount: number;
    }
    ```
  - `getComparisonStats()` API.
  - 통계 갱신 시 main.ts에 알리기 위해 message로 push (webview→webview 같은
    쪽 hook, 또는 컴포넌트가 직접 read). main.ts가 setFrame 후 stats 갱신
    호출.
- `src/editor/crystalEditorProvider.ts`:
  - phases-section 안 또는 trajectory-section 아래에 신규 `comparison-stats`
    div. RMSD + max/mean/p95 + matched/unmatched 표시.
- `src/webview/main.ts`:
  - `updateComparisonStatsUI()` — renderer.getComparisonStats() 호출 +
    text content 갱신.
  - setFrame 또는 setTrajFrame 후, compareToPhase/clearComparison 후 호출.

**통과 기준:**
- Comparison 활성 시 stats 표시.
- frame 변경 시 즉시 갱신.
- clearComparison으로 stats 영역 사라짐.

**작업 단위 추정:** 0.5d.
