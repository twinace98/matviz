## 17.3.1 — `--frame N` 단일 frame 추출

**파일:**
- `scripts/render.ts`:
  - `RenderOptions.frame: number` 필드 (default 0).
  - `--frame <n>` 플래그 parser.
  - help text 업데이트.
  - `parseStructureFile` 호출을 trajectory-aware로 변경: 파일이
    `parseStructureFileTraj`로 multi-frame trajectory 반환하면
    `trajectory.frames[clamp(N, 0, frameCount-1)]`을 사용. 단일 frame이면
    무시.
- `skills/matviz-render/SKILL.md` + `README.md` — `--frame` 플래그 설명 +
  trajectory 형식별 예제.
- `test/visual/fixtures/xdatcar-frame3/scene.json` (신규) — XDATCAR 5-frame
  + frame=3 → 4번째 frame 렌더.
- `test/visual/baseline/xdatcar-frame3.png` — initial baseline.
- `test/visual/harness.ts` — SceneConfig에 `frame?: number` + buildArgs 매핑.

**Dispatch logic:**
```ts
import { parseStructureFile, parseStructureFileTraj } from '../src/parsers/index';

// First try trajectory-aware path; if multi-frame, pick OPTS.frame.
const trajResult = parseStructureFileTraj(content, filename);
let structure;
if (trajResult.trajectory.frames.length > 1) {
  const N = trajResult.trajectory.frames.length;
  let idx = OPTS.frame ?? 0;
  if (idx < 0 || idx >= N) {
    console.warn(`--frame ${idx} out of range [0..${N-1}], clamping`);
    idx = Math.max(0, Math.min(idx, N - 1));
  }
  structure = trajResult.trajectory.frames[idx];
} else {
  structure = trajResult.trajectory.frames[0];
}
```

**통과 기준:**
- xdatcar-nve fixture + `--frame 3` → frame[3] (Na drift x = 0.03×5.64 ≈ 0.169Å)
  렌더. baseline 매칭.
- 비-trajectory fixture + `--frame N` → frame 0과 동일 (warn 포함).
- 범위 외 → clamp + warn.

**작업 단위 추정:** 0.5d.

## 17.3.2 — `--all-frames` 이미지 시퀀스 렌더

**파일:**
- `scripts/render.ts`:
  - `RenderOptions.allFrames: boolean` (default false).
  - `--all-frames` 플래그 parser.
  - help text 업데이트.
  - main 함수: `--all-frames` 활성 시 trajectory 파일에 대해 N번 loop —
    각 frame마다 OPTS.frame=i 설정 + 출력 파일명 templating
    (`<base>_NNNN.png`, 4자리 zero-padded 또는 5자리 if N>9999).
  - `--all-frames` + `--frame N` 동시 → `--all-frames` 우선 + warn.
- `skills/matviz-render/SKILL.md` + `README.md` — `--all-frames` 예제 +
  PNG 시퀀스 → ffmpeg gif 워크플로우 노트.

**Filename templating:**
```ts
function frameOutputPath(base: string, frameIdx: number, totalFrames: number): string {
  const width = totalFrames > 9999 ? 5 : 4;
  const padded = String(frameIdx).padStart(width, '0');
  // Replace `.png` extension with `_NNNN.png`
  return base.replace(/\.png$/i, `_${padded}.png`);
}
```

**Loop strategy (J7 simplification):**
첫 컷에서는 `--all-frames` 활성 시 main이 자체적으로 N번 invocation 시뮬레이션
(각 frame마다 별도 Puppeteer page.goto + screenshot). v0.17.x에서 browser
재사용 + page.evaluate로 4× 가속.

**통과 기준:**
- xdatcar-nve fixture + `--all-frames -o /tmp/test.png` → /tmp/test_0000.png
  ~ /tmp/test_0004.png (5개) 생성.
- `--all-frames --frame 2` → warn + all-frames 진행.
- 비-trajectory + `--all-frames` → `<base>_0000.png` 1개 출력.

**작업 단위 추정:** 0.5d.
