## 15.1 — Sphere impostor

**파일:** 신규 `src/webview/shaders/sphereImpostor.glsl.ts` (vertex + fragment 문자열), `src/webview/atomRenderer.ts` (or `renderer.ts` 내부). `shared/elements-palette.ts`에서 radius 전달 유지.

**접근:**
1. 각 원자당 카메라 방향 billboard quad 1개 instanced. instanceMatrix에 center/radius 인코드.
2. Fragment shader: ray-sphere intersection → depth write + Phong-lite (ambient + directional).
3. `gl_FragDepth` 사용 — depth 정확도 유지 (다른 지오메트리와 올바른 z-order).
4. `renderer.setCapabilities().isWebGL2` 확인 후 fragment depth 확장 가용성 체크.
5. 사이드 패널 Style 섹션에 토글 ("Impostor atoms"); 기본값: 원자 수 ≥ 2000이면 자동.

**통과 기준:**
- 16 k 원자 stress fixture에서 `renderer.info.render.triangles` 10× 감소.
- 외곽 픽셀이 32-segment SphereGeometry와 육안 구분 불가 (동일 카메라 스크린샷 diff < 2 %).
- Selection highlight (현재 `HIGHLIGHT_COLOR`) 정상 동작.

## 15.2 — GPU 피킹 ✅

**파일:** `src/webview/picking.ts` (신규), `renderer.ts` (분기).

**구현:**
1. `AtomPickingRenderer`: 1×1 `WebGLRenderTarget` + mirror pick scene. 
   - Impostor 원자는 ray-sphere 테스트 + `gl_FragDepth` 기록 (billboard quad가 아닌 구 표면으로 픽킹).
   - non-impostor 원자는 저-poly sphere geometry로 단순 ID 패스.
2. `iPickColor` per-instance 속성에 `globalIdx+1`을 RGB24로 인코드 (0 = no hit).
3. `camera.setViewOffset(fullW, fullH, pxX, pxY, 1, 1)` 로 클릭 픽셀만 렌더.
4. `GPU_PICK_THRESHOLD = 5000` — 이하는 기존 raycaster 경로 유지.
5. `rebuild()` 는 `buildVisuals()` 마지막, `setOrtho()` 는 `setCameraMode()` 에서 호출.

**통과 기준:**
- 50 k 원자 구조 (임시 슈퍼셀)에서 클릭 응답 < 5 ms (`performance.now()` 측정).
- 기존 선택/측정 동작 회귀 없음.

## 15.5 — Cylinder impostor (bond)

**파일:** 신규 `src/webview/cylinderImpostor.ts`; `src/webview/bondRenderer.ts` 확장.

**접근:**
1. `InstancedBufferGeometry` — billboard quad (4 verts, 2 tris) + per-instance attrs: `iPosA` (vec3), `iPosB` (vec3), `iRadius` (float), `iColorA` (vec3), `iColorB` (vec3).
2. VERT: bond 축 `d = posB − posA` 정규화, view-space 카메라 ray 와 수직이면서 축과도 수직인 perpendicular basis 계산 → billboard quad 를 `(axis, perp)` 평면에 배치하고 `radius` 만큼 측방향 + `radius` 만큼 축방향 padding 포함. 이렇게 하면 fragment ray 가 실린더 표면과 교차할 수 있는 bounding envelope 확보.
3. FRAG: world 또는 view-space ray-cylinder 교차. 축에 평행한 ray 는 discard. 교차점 `t = hitOnAxis` 를 `[0, bondLength]` 로 클램프 (cap 없음 — 끝은 sphere impostor 가 덮음). bicolor 는 `dot(hit - mid, axis) < 0` 이면 colorA 아니면 colorB.
4. `gl_FragDepth` 로 z-order 정확. sphere impostor 와 동일하게 `uniform mat4 projectionMatrix;` 를 FRAG 에 명시 (three.js fragment prefix 는 projectionMatrix 를 자동 주입하지 않음).
5. BondRenderer: `BondRendererDeps` 에 `getImpostorEnabled: () => boolean` 추가. `rebuild()` 에서 `bondStyle !== 'line' && displayStyle !== 'wireframe' && getImpostorEnabled()` 면 impostor 경로, 아니면 기존 cylinder 경로. unicolor 는 `iColorA=iColorB=unicolorColor` 로 동일 path 재사용.
6. `renderer.setImpostorEnabled()` 는 atoms + bonds 둘 다 재빌드 (이미 `buildVisuals()` 호출하므로 bondRenderer.rebuild 도 같이 트리거됨).
7. raycast(): no-op (bond 피킹은 현재 UI 에 없음).
8. 체크박스 라벨 `Use sphere impostor` → `Use impostor rendering` 으로 rename (sphere + cylinder 둘 다 제어하므로).

**통과 기준:**
- `stress-20k.poscar` 에서 `renderer.info.render.triangles` 가 cylinder 분량만큼 추가 감소 (sphere impostor 와 같은 비율).
- bicolor / unicolor / line 3종 시각적으로 real cylinder 와 구분 불가 (중간 경계 선명, 끝단은 sphere impostor 가 덮어줌).
- 팔레트 토글 및 bond style 스위치 50회 반복 시 geometry leak 없음.

## 15.4 — WebGPU 평가 ❌ 기각 (2026-04-18)

**판정:** 프로토타입 진입 없이 비용/편익 검토 후 기각.

**근거:**
- Three.js `WebGPURenderer` 는 raw GLSL `ShaderMaterial` 을 지원하지 않음 → 커스텀 impostor/picking shader 3종을 **TSL (Three Shading Language) Node material** 로 재작성 필요. 작업 규모 1~2일.
- 현재 matviz 는 이미 15.1 sphere impostor + 15.3 chunked frustum culling + 15.5 cylinder impostor 로 draw call 및 삼각형 수가 최적화 — WebGL2 대비 WebGPU 의 **즉각적 성능 이득이 불분명**. 컴퓨트 셰이더 기반 재설계 (GPU bond detection, GPU picking grid, GPU marching cubes) 가 들어가야 WebGPU 의 진짜 차이가 드러나며, 이는 v0.15 스코프 밖.
- CLI 렌더러 (Puppeteer + SwiftShader) 는 WebGL2 전용 — WebGPU 로 전환 시 headless 소프트웨어 fallback 부재 (Dawn/Vulkan 필요) → 웹뷰/CLI 경로 이원화 리스크.
- VSCode 웹뷰 Chromium 버전 지원 불균등 (구버전 사용자 영향).

**재평가 조건 (언제 돌아올 만한가):**
1. 50 k+ 원자 구조에서 WebGL2 경로가 draw call/CPU 오버헤드로 병목이 확인될 때 (현재 미확인).
2. 컴퓨트 기반 재설계 (bond/picking/isosurface) 가 로드맵에 들어올 때 — v0.17~v0.18 전후.
3. Three.js WebGPU API 가 안정화 되고 TSL 에코시스템이 성숙할 때 (2026 후반~2027).

**후속 조치:** 없음. `Plan.md` v0.15.4 및 결정 게이트 항목 갱신만.
