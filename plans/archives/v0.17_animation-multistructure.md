# v0.17 — Animation & Multi-structure

**목표:** MD 시뮬레이션 결과 (XDATCAR / AXSF / extended XYZ trajectory)를 매트릭스
시각화에 도입. v0.16의 정적 fancy-rendering 위에 *시간 차원*을 더해 1k+ atoms ×
1k+ frames 재생을 실시간 처리. 부수적으로 multi-phase overlay (두 structure 동시
표시) + comparison mode (대응 atom 변위 벡터).

**전제 컨텍스트:** v0.16 모든 기능 (16.0~16.5 + 4 CLI ports + magmom black-fix
patch) 이미 main에 있음. 현재 `package.json` 0.15.2 (v0.16 미릴리즈). v0.17 작업은
post-v0.16 main에서 진행 — v0.16.0 정식 릴리즈는 사용자 승인 후 별도 진행. v0.17
commit history는 v0.16 위에 누적.

**상태:** 킥오프 초안 — 승인 후 **완전 자율 진행**.

## 결정 게이트 (사전 채택 — 자율 진행 중 별도 사용자 승인 불필요)

10개 게이트 사전 채택:

1. **CrystalTrajectory 타입 모양**: 별도 wrapper.
   ```ts
   interface CrystalTrajectory {
     frames: CrystalStructure[];      // length ≥ 1 (single-frame = length 1)
     latticeMode: 'fixed' | 'per-frame';
   }
   ```
   기존 `CrystalStructure` 계약 보존. v0.16 옵셔널 필드 패턴 (per-atom array)이
   trajectory에는 자연스럽지 않음 (frame은 *atom* 차원이 아니라 *시간* 차원).

2. **Parser API 정책**: 두 entry point.
   - 기존 `parseStructureFile(): CrystalStructure` — 첫 frame만 반환 (모든 호출
     사이트 마이그레이션 부담 회피, 기존 단발 시각화는 변동 0).
   - 신규 `parseStructureFileTraj(): CrystalTrajectory` — multi-frame 필요한
     호출 사이트 (재생 UI, multi-phase, comparison)에서 사용.
   - 단발 호출 사이트가 multi-frame 파일을 마주치면: parsers가 첫 frame만 반환,
     UI에서 "this file has N frames — open trajectory mode" 토스트 (사용자
     opt-in).

3. **재생 모드 진입**: 사이드 패널 "Trajectory" 섹션이 *데이터 기반 표시*
   (frames.length > 1일 때만). v0.16 ellipsoid/partial/magmom section 패턴과
   동일.

4. **Multi-frame 우선순위**: AXSF (단순) → XDATCAR (사용자 핵심) → extended XYZ
   (ASE 호환).

5. **XDATCAR 변수 셀 (NPT) 처리**: "Direct configuration=" 라인 감지 — 변수
   셀 모드면 latticeMode='per-frame', 각 configuration block 헤더에서 lattice
   재추출. NVE는 latticeMode='fixed', 첫 lattice를 모든 frame에 inherit.

6. **Bond 재계산 default off**: 재생 중 매 frame O(N) bond detection은 1k+
   atoms에서 30fps 미달. UI 토글 "Recompute bonds every frame" + warning hint.
   Default off 시: 첫 frame의 bond pattern을 모든 frame에 inherit (변동 셀에서
   bond가 부정확할 수 있지만 시각적으로 acceptable).

7. **Memory budget**: raw frames in-memory if N×frames ≤ 10⁷ (≈ 130 MB peak
   for typical position+species). 초과 시 console.warn + 사용자에게 "open as
   text" 권장. Streaming은 v0.17.1.

8. **Interpolation policy**: 없음 — frame-stepped only. 인접 frame이 멀어
   playback이 jagged하면 사용자가 frame을 더 dense하게 export. v0.18+
   재논의.

9. **17.2 multi-phase 두 번째 구조 입력 방식**: 명령 팔레트 `MatViz: Add Phase`
   → file picker → 동일 viewport에 추가 (offset 명시 또는 cell-overlap). 두
   번째 phase도 trajectory 가능 (NVE 두 시뮬 비교 등).

10. **17.3 사전 분리 결정**: v0.17.0 scope에서 제외 (사용자 결정 2026-04-23).
    v0.17.1 별도 plan 쌍에서 처리. atom matching 알고리즘 / displacement
    vector 렌더 / UI는 v0.17.1 kickoff 시 구체화.

## 로컬 commit 정책 (v0.16에서 carry-over)

- 각 .x 피처(17.1.0, 17.1.1, …, 17.3) = **최소 1 checkpoint commit** (사용자
  `git checkout`/`git revert` 가능).
- 자연스러운 sub-commit 분할 가능 (각자 독립 검증 통과 — bisect-safe).
- Pre-commit hook 실패 시 root cause fix 후 NEW commit (--amend 금지).
- **Push는 여전히 사전 승인** (외부 가시).
