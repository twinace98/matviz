## 결정 게이트 (사전 채택)

8개 게이트:

1. **Phases UI 구조**: `phase-section` 신설, secondary phases 목록 + 각 row
   controls (visibility checkbox, opacity slider, remove button).
   Comparison toggle은 phase list 아래 별도 row.
2. **Phase 추가 entry**: 사이드 패널 "Add Phase…" 버튼이 명령 팔레트와 동일
   동작 (file picker 호출). 명령 팔레트도 유지.
3. **재생 키보드**: Space → toggle play/pause. Webview focus 우선 (input
   focus 시 비활성). other shortcuts (arrow-left/right for prev/next frame)
   는 첫 컷 보류.
4. **Speed slider**: 0.25× ~ 2× 범위, 기본 1× (30fps). slider step 0.25.
   30fps × multiplier로 frame 진행 속도 조절.
5. **Once-only loop**: toggle "Loop" (default on). off 시 마지막 frame에서
   자동 일시정지.
6. **Frame number direct input**: frame label을 input field로 변경 (현재
   `1 / 4` → `<input value=1> / 4`). Enter로 jump.
7. **PBC minimum-image**: matchByNN의 `lattice?` 옵션이 있으면 displacement
   d 계산 시 d → d - round(d_frac)·L (각 axis별). lattice ref 둘 다 동일할
   때만 활성 (`primaryLat === secondaryLat` ref equality, fixed-cell case).
8. **RMSD 공식**: matched pairs의 displacement L2 norm² 평균의 sqrt.
   Unmatched는 RMSD 계산에서 제외 (별도 카운트만 표시). Per-frame 갱신은
   recomputeComparison에서 결과 cache + main.ts UI 갱신.
