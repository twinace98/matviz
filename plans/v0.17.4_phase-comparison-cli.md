# v0.17.4 — Multi-Phase + Comparison CLI Exposure

**목표:** v0.17.0의 multi-phase overlay (17.2) + v0.17.1의 comparison
displacement vectors (17.3)를 headless CLI 렌더러에 노출. matviz-render
skill이 webview/CLI 양쪽에서 모든 v0.17 기능을 사용할 수 있게 됨 → 보고서
워크플로우에서 "before/after MD" 또는 "relaxed vs unrelaxed" 비교 그림을
명령 한 번으로 생성 가능.

**전제 컨텍스트:** v0.17.3 묶음 릴리즈 (commit `020e65a`, tag `v0.17.3`,
push origin + sogang-qmp 완료) 직후 main에서 시작. 다음 묶음 릴리즈는
v0.17.4 closure 시점 (또는 v0.18 진입 시) 사용자 결정.

**상태:** 킥오프 초안 — **즉시 자율 진행** (사용자 4 결정 carry-over).

## 범위

| # | 피처 | 통과 기준 |
|---|------|-----------|
| 17.4.1 | `--phase <file>` 플래그 (반복 가능) — secondary structure overlay | `--phase nacl.cif` 추가 시 primary 위에 transparent secondary 렌더; 다중 `--phase` 누적 (`--phase a.cif --phase b.cif` → 2개 secondary) |
| 17.4.2 | `--compare-to-phase` 플래그 — 첫 phase와 NN displacement 화살표 | `--compare-to-phase` 활성 시 변위 화살표 (Viridis colormap); RMSD/displacement summary는 stdout 출력 (CLI에는 사이드 패널 없으므로); PBC-aware (primary와 secondary가 같은 lattice면) |

**범위 외 (v0.17.x 또는 v0.18+로 deferred):**
- Per-phase opacity / offset CLI 옵션 (`--phase file.cif:opacity=0.3:offset=2,0,0`)
  — syntax 복잡, 현재 default 0.5 + (0,0,0)으로 충분.
- Comparison colormap 선택 (`--displacement-colormap`) — Viridis fixed
  (webview도 동일).
- Comparison summary JSON 출력 옵션 (`--summary-json`) — stdout text로 충분.
- Frame-aware comparison CLI — `--all-frames + --compare-to-phase` 조합은
  의미 모호 (각 frame마다 같은 secondary 비교? secondary가 frame i면? 등).
  v0.17.x에서 syntax 정착 후 추가.

## 결정 게이트 (사전 채택)

7개 게이트:

1. **`--phase <file>` 반복 가능**: 명령행에 여러 번 등장 시 누적 (parseArgs
   에서 array push). webview의 secondary phases array와 매핑.
2. **Default offset `(0,0,0)` + opacity `0.5`**: webview default와 동일.
   사용자 customization은 v0.17.x로 deferred.
3. **`--compare-to-phase` flag-only (인자 없음)**: 첫 `--phase`와 비교 (multi-
   secondary 선택은 webview처럼 first only). 명령행에서 어느 phase 비교할지
   지정은 v0.17.x.
4. **`--compare-to-phase`만 지정 시 (without `--phase`)**: error + exit 1
   (webview는 toast warn; CLI는 사용자 명령 의도 명확이라 hard error).
5. **PBC 자동 활성**: webview 17.2.2 와 동일 — primary와 secondary가 같은
   lattice이면 minimum-image. CLI는 lattice ref 비교 안 됨 (각 invoke가
   별 process이라 ref equality 무의미) → lattice 값 element-wise 비교로
   동등성 판정.
6. **`--all-frames + --compare-to-phase` 조합**: deferred — error + exit 1
   ("not supported in v0.17.4; use --frame N + --compare-to-phase per frame
   or run sequentially").
7. **Comparison summary stdout**: `--compare-to-phase` 활성 시 stdout에
   `RMSD: 0.123 Å (matched 50, unmatched 0, max 0.456 Å, mean 0.321 Å,
   p95 0.412 Å)` 한 줄 출력. user가 `2>/dev/null` 또는 grep으로 필터링
   가능. JSON 출력은 후속 옵션.
