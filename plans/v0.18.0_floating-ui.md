# v0.18.0 — Floating UI (V2 redesign)

## Goal

Adopt the **V2 Floating Panels** design language (handoff bundle, 2026-04-23
chat) for the MatViz webview. The 3D viewport becomes the full canvas; chrome
recedes into translucent glass panels that float above it. Replaces the current
edge-attached opaque chrome wholesale.

## Why now

User picked V2 after seeing four design directions (V1 Refined Classic, V2
Floating Panels, V3 Command-First, V4 Editorial Split). The chat shows the
direction was committed ("V2로 결정할게") and then iterated through ten polish
rounds — measure HUD, tabs, info pill, help layout, switch styling, alignment.
This version implements the locked-in V2 design as production code.

## Roadmap impact

`Plan.md` previously pencilled v0.18 as "Editor integration (split-pane, VSCode
settings namespace, undo/redo)" — that slot was a placeholder, not a started
version. Editor integration **shifts to v0.19**. No work is lost; v0.19's plan
will inherit the editor-integration backlog when it kicks off.

## Scope

**In scope:**
- Replace `media/styles.css` chrome with V2 glass-tokenized stylesheet
- Restructure `crystalEditorProvider.ts` HTML: floating toolbar (centered),
  floating side panel (rounded), info pill (bottom-left), axis gizmo
  (bottom-right), restyled help overlay
- iOS-style toggle switches (VSCode blue) replacing all visibility checkboxes
- Style chips replacing the `<select>` for display style
- V2 supercell stepper (`− N +` horizontal, no upper bound) replacing the V1
  ▲/▼ stepper for sc-a/b/c
- Measure HUD overlay (top-right): kind switcher, hero distance value, atom-pair
  card, Δ fractional/cartesian rows, footer hint + copy
- Fold in this session's already-built **unified inline-SVG icons** and
  **step-angle / step-zoom numeric steppers** as Feature 1 (groundwork)

**Explicitly out of scope:**
- **In-webview tabs bar** — V2 includes one, but matviz uses VSCode tabs (one
  webview per file). Skipped; revisit if/when matviz grows multi-file workflow.
- **Empty state drop zone** — `openCustomDocument` throws on parse failure,
  so the webview never reaches an empty state. V2 mockup not reachable.
- **V2's icons and fonts** — explicitly excluded by user. Keep V1 unified
  SVGs (this session) and `--vscode-font-family` monospace.
- **Light-theme alpha tuning for backdrop-filter** — V2 was authored against
  dark VSCode. Ship dark-tuned values; if light-theme legibility regresses,
  open a follow-up rather than block this version.

## Decision gates

1. **Gate-A — Kickoff approval** (this doc + impl spec): user signs off on
   scope (skip tabs/empty) and on rerouting v0.18 from editor-integration to
   floating UI. **Pass criterion:** explicit "approved" from user.
2. **Gate-B — Visual review after Feature 6 (info pill / gizmo / V2
   supercell)**: user opens a fixture in VSCode and confirms the floating
   chrome reads correctly on a non-trivial structure (e.g. SrTiO3 supercell
   with polyhedra on). **Pass criterion:** user OK to continue to Measure HUD.
3. **Gate-C — Pre-release**: type-check + build clean, all 4 fixture types
   smoke-tested (CIF, POSCAR, CHGCAR with iso, AXSF trajectory). User signs
   off on cutting v0.18.0. **Pass criterion:** explicit release approval.

## Risk register

- **CSP / inline-style risk**: webview CSP is nonce-only (`'unsafe-inline'`
  dropped in v0.13.1). All new styles go in `media/styles.css`; any inline
  `style="..."` added during this version must be flagged and refactored.
- **`backdrop-filter` on light VSCode themes**: alpha layers tuned for dark
  may wash out on light. Mitigation: `prefers-color-scheme: light` overrides
  if regression observed; otherwise document as known limitation.
- **Picking-state coupling for Measure HUD**: HUD reads selection from
  `picking.ts`. Need a clean signal (event/observer) so HUD doesn't poll —
  details land in the impl spec.
- **localStorage state shape change**: V2 supercell loses upper bound (was 5).
  Existing saved state (`schemaV1`) stays compatible (numbers in range still
  parse), but values >5 from new sessions will need to load correctly into a
  later version that re-introduces a max — flag in impl spec.
