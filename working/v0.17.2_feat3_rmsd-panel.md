## Autonomous judgments recorded

### J1. RMSD over matched pairs only

unmatched (threshold 초과 또는 same-species 부재) atom은 RMSD 계산에서 제외.
사용자에게는 "matched / unmatched" 카운트 별도 표시 → unmatched가 많으면
RMSD 신뢰도 낮음을 user가 판단 가능. 다른 정의 (unmatched에 penalty 추가)는
워크플로우에 따라 의미 다름 → 가장 단순한 정의 채택.

### J2. p95 magnitude (sort 후 95-percentile)

표준 RMSD 통계 보완. `mags[Math.floor(0.95 * N)]` (정렬된 N개 중 0-based
인덱스). 큰 N (가까운 atom 다수)에서 outlier 검출에 유용.

### J3. fmt heuristic

값이 0.001 미만이면 "<0.001" (precision 표시 무의미), 그 외엔 `toPrecision(3)`
(3 sig fig). 일반 변위 범위 0.001–10 Å에 적합.

### J4. setTrajFrame이 stats UI 갱신 트리거

renderer.setFrame이 `recomputeComparison()`을 자동 호출하므로 lastStats가
이미 갱신됨. main.ts setTrajFrame은 단순히 UI re-render. 명시 함수 분리로
side effect 흐름 명확.

### J5. clearComparison이 stats null

stats panel이 깔끔하게 사라짐 (UI hidden). user가 "comparison off" 상태에서
이전 통계 잔재 보지 않음.

### J6. Unmatched=0 → 풀 카운트 비교

같은 종/배치의 두 결정 비교에서 unmatched=0이 일반. unmatched>0은 (a)
species 다름, (b) atom 위치가 threshold 초과 변동, (c) atom 갯수 다름 신호.
panel display로 즉각 인식.
