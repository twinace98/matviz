# v0.13.1 · Feature 1 — Shared elements-data

**Status:** complete
**Date:** 2026-04-17

## Goal

Eliminate 3-way element data drift (Node parser table 80 elements, browser webview table 68 elements, CLI renderer inline table ~30 elements) by establishing a single source of truth.

## Changes

### Created
- `src/shared/elements-data.ts` — masterset (80 elements) with `ElementData { symbol, number, color, covalentRadius, vdwRadius, displayRadius }`. Exports `ELEMENTS`, `DEFAULT_ELEMENT`, `getElement(symbol)`, `getElementByNumber(n)`. Filled in 12 previously-missing lanthanide `displayRadius` values (Pr–Lu).
- `src/shared/elements-palette.ts` — `ColorPalette` type, `brighten()` HSL-lift, `getElementPaletteColor()`, `getPaletteLineColors()`.
- `test/fixtures/nd2o3-hex.cif` — Nd2O3 A-type hexagonal (P-3m1, 5 atoms). Exercises a lanthanide that previously fell back to defaults in the webview/CLI.

### Deleted
- `src/parsers/elements.ts`
- `src/webview/elements-data.ts`

### Edited
- `src/parsers/types.ts` — dropped `ElementData` interface (moved to shared).
- `src/parsers/cubeParser.ts`, `src/parsers/xsfParser.ts` — import from `../shared/elements-data`.
- `src/webview/renderer.ts` — imports from shared modules; `getWebElement` → `getElement`.
- `scripts/render.ts` — builds a compact `CLI_ELEMENTS` map from the shared table at Node-time and inlines it into the HTML template via `JSON.stringify`, replacing the hand-maintained inline table.

## Validation

- `npx tsc --noEmit` → no errors
- `npm run build` → Build complete
- CLI smoke tests:
  - `node dist/render.js test/fixtures/nd2o3-hex.cif -o /tmp/nd2o3.png --supercell 2,2,1 --view std` → 145690 bytes
  - `node dist/render.js test/fixtures/nacl.cif -o /tmp/nacl.png --supercell 2,2,2 --view std` → 155032 bytes (regression)

## Notes

The CLI renderer's HTML template runs inside Puppeteer (browser), so it cannot directly import TypeScript modules. The bridge is Node-side construction of a compact `{color, cr, vdw, dr}` shape followed by `JSON.stringify` interpolation. This preserves the existing browser-side code path while making the shared table the sole authority.
