## Autonomous judgments recorded

### J1. 인터페이스 형식 — vectors는 행 vs 열

표준 Jacobi 출력은 V matrix의 *컬럼*이 eigenvector. 그러나 caller (16.1 (3/3)
ellipsoid 렌더러)는 `vectors[i]`로 i번째 eigenvector에 직접 접근하길 원함.

**선택:** `vectors[i]: Vec3` (i번째 eigenvector를 row로 반환, values[i]와
페어링). reconstruction 공식은 `M[i][j] = Σₖ vectors[k][i] · values[k] ·
vectors[k][j]` (test-symeigen.ts에 명시 + 검증).

trade-off: matrix-multiply 시 transpose가 필요할 수 있지만, 16.1 ellipsoid
변환에서는 `T(c) · R · diag(r₁,r₂,r₃)` 형식의 4×4 매트릭스를 element-wise로
구성할 것이므로 row 형식이 더 자연.

### J2. Convergence threshold + max sweep

`MAX_SWEEPS = 50` + `TOL = 1e-10` (off-diagonal sum-of-squares).
- 일반 Uᵢⱼ (well-conditioned, 작은 off-diag)는 ≤ 6 sweep에 수렴.
- Edge case (near-degenerate eigenvalues, e.g. Uᵢᵢ ≈ Uⱼⱼ + tiny off-diag)도
  test 4 case 3에서 검증 (`[[10,0.001,0],[0.001,10,0],[0,0,5]]`).
- 50 sweep에서 미수렴 시 silent return — physical Uᵢⱼ에서는 발생 안 한다고
  가정. 발생 시 결과는 best-effort.

### J3. 정의 부호 — physical Uᵢⱼ는 PSD

물리적 thermal motion tensor는 positive semi-definite (eigenvalues ≥ 0).
수치 drift로 작은 음수 eigenvalue가 나올 수 있음. **솔버 자체는 클램프하지
않음** — 책임은 caller (16.1 (3/3) 렌더러). 16.1 (3/3) 코드는
`max(0, eigenvalue)` 적용 + 콘솔 warning 예정 (plan §16.1 명시).

### J4. 검증 fixture 5 (실제 O 원자) tolerance 1e-7 → 1e-6

첫 시도에서 1e-7 사용 → fail. 이유: Jacobi convergence threshold가 1e-10인데
이것은 *off-diagonal sum-of-squares* 임계값이고 reconstruction 정확도와 직접
대응 안 함. typical 5–6 sweep 후 reconstruction 잔차는 ~1e-9 정도이지만 모든
입력에 보장 안 됨. 1e-6으로 완화 (다른 테스트와 일관).

### J5. 외부 의존성 도입 안 함 (sub-commit 결정 게이트 #3 채택안)

대안:
- ml-matrix (npm package, full eigen suite, ~50KB)
- numeric.js (general numerics, ~150KB)
- 직접 구현 (Jacobi 3x3, ~90 LOC) ✅

**선택 근거:** 3×3 symmetric은 직접 구현이 stable + 의존성 추가 회피. webview
번들 크기 영향 0. 모든 sub-commit이 *각자* dependency-clean.

## Known limitations / deferred

1. **Non-convergence handling**: 50 sweep에서 미수렴 시 silent return.
   Pathological 입력 (만약 발생 시) 콘솔 warning 추가 권장 → (3/3) 또는
   16.x.
2. **Eigenvalue 부호 보정 (PSD 강제)**: caller responsibility. (3/3) 렌더러가
   `Math.max(0, λ)` + warning 적용 예정.
3. **확장 (4×4, 일반 비대칭)**: 현재 3×3 sym 전용. 다른 케이스 필요 시
   별도 함수 또는 외부 라이브러리 도입 검토 — 16.x.
4. **Mat3 → Three.js Matrix3 어댑터**: 없음. caller가 직접 변환 (간단). 필요
   시 `toThreeMatrix3()` 추가 가능.
