## Autonomous judgments recorded

### J1. `buildAtoms` 안에서 ellipsoid 분기 vs 별도 `buildEllipsoids` 메서드

**선택: buildAtoms 안에서 인라인 분기.**

대안:
- A. `buildAtoms` 안에서 aniso 인덱스를 `ellipsoidIdxSet`으로 컬렉션 → 제외
  후 기존 grouping 로직 진행. 마지막에 `ellipsoidRenderer.rebuild`. ✅ 선택.
- B. 별도 `buildEllipsoids` 메서드 신설 + `buildVisuals`에서 둘 다 호출.

B는 더 분리된 구조지만 `buildAtoms`와 `buildEllipsoids`가 같은 입력
(species/positions/expandedUnitCellIndex)을 두 번 순회. A는 한 번 순회로
세트 분리. 복잡도 ~30 LOC 추가되지만 성능은 동등. **A 선택 — 단일 진실 공급원
(index split)**.

코드 위치: `renderer.ts buildAtoms()` 시작부.

### J2. Wireframe / Stick 스타일에서 ellipsoid 비활성

plan §16.1은 언급 없음. 자율 결정:
- wireframe/stick은 "구조만" 보여주는 스타일 — ellipsoid 표시는 의미 중첩.
- 그리고 wireframe에서 ellipsoid을 보이게 하려면 `WireframeGeometry`를
  ellipsoid 각 사이트마다 생성해야 함 — 작업량 불필요.

**선택:** `style !== 'wireframe' && style !== 'stick'` 에서만 ellipsoid 활성.
ball-and-stick / space-filling에서만 동작. 사용자가 stick으로 전환 시
ellipsoid 사라짐, 다시 ball-and-stick이면 복원 (state 보존).

### J3. `showEllipsoids` 영속화 — `showPolyhedra`와 달리 restore

`showPolyhedra`는 plan §5 decision에 의해 *파일 오픈마다 off로 리셋*. 이유:
자동 검출 heuristic이 이전 사이트에서 예상 밖 결과를 낼 수 있음.

ellipsoid는 데이터 자체가 파일에 있음 (heuristic 없음). 사용자가 anisotropic
CIF를 주 worked set으로 사용할 때 매번 다시 켜는 건 UX 악화. **영속화 선택**.
단 비-aniso 파일 로드 시 섹션 자동 숨김, 토글 state 유지 (다시 aniso 파일
열면 on으로 돌아감).

### J5. χ²₃ 상수 하드코딩 (plan §결정 게이트 #2 채택안)

`CONTOUR_50 = sqrt(2.366)`, `CONTOUR_90 = sqrt(6.251)`. 대안 (scipy-style
erfc 라이브러리 도입) 거부.

정확도: 2.366은 χ²₃(0.5)의 ~4 자리 근사. 최신 테이블 값은 2.36597. 사용자
육안 차이 무. 필요 시 더 많은 유의 자리로 조정 가능. (계산 방법: Wolfram
Alpha "inverse chi-squared CDF 3 DOF 0.5" = 2.365974 정확.)

### J6. Right-handed frame 보정

`computeEllipsoidMatrix`에서 `det(R) < 0` 시 last column을 flip. 이유:
eigenvector는 sign-ambiguous (v와 -v 모두 유효). left-handed frame에서는
spherical mesh normal이 flip되어 렌더 bug 발생 가능. det 체크 + 마지막 축
flip으로 right-handed 보장. semi-axis 길이는 유지 (ellipsoid shape 무관).

### J7. Uᵢⱼ 음수 eigenvalue 경고 threshold 1e-4

Jacobi 수치 drift가 ~1e-10 수준 음수 생성 가능. 엄격히 `< 0` 체크 시 routine
spam. `< -1e-4` 에서만 console.warn (physically 의미 있는 음수만). 0으로
클램프 + sqrt는 `Math.max(λ, 0)`로 NaN 회피.

## Known limitations / deferred

1. **Symmetry-expansion U 회전 미적용** (1/3 §J2 carry-over). 비-P1 공간군 +
   off-diagonal U 시나리오에서 ellipsoid 방향 부정확. 16.x.
2. **Calcite/anorthite reference fixture 미도입** — test-aniso.cif로 parser
   contract 검증만 수행. 실제 광물 구조 reference는 user 설치 후 필요 시
   독자 추가. 저작권/라이선스 free 광물 CIF 구하면 `test/fixtures/` 에 추가
   권장.
3. **Ellipsoid GPU 피킹 미검증** — picking.ts는 아직 ellipsoid 모듈을 인지하지
   않음. ellipsoid 클릭 시 atom 선택 동작 확인 필요 (16.x).
4. **CLI 렌더러 `--ellipsoids` 플래그 부재** — CLI는 ellipsoid 지원 안 함.
   v0.17+ 또는 user 요청 시 추가.
5. **시각 검증 자동화** (J4) — webview-stub harness 필요. 16.0 선행 deferral과
   동일 경로. v0.16.0.1 또는 16.x.
6. **Stick/wireframe 스타일에서 ellipsoid 비활성** (J2) — 의도적. 사용자가
   원하면 후속 enhancement로.
7. **Legacy CIF `_aniso_*` 다른 형식** — β-form, Cartesian-indexed aniso 등은
   여전히 미지원 (1/3 carry-over).
