## Autonomous judgments recorded

### J1. minimum-image는 NN 검색 자체에 적용

대안 (검색은 raw + displacement만 보정)은 cell boundary 가까운 atom이
매칭 누락. 검색 단계에서 PBC 적용 → boundary atom이 다른 cell의 image와
정확 매칭. 비용은 per-check minimum-image 계산 (~10 산술 연산) — 5k atom
N²=25M ops × 10 = 250M, 0.5초 미만.

### J2. Spatial bin은 PBC mode에서 비활성

bin + PBC 결합은 neighbor bin wraparound (cell boundary 가까운 atom이
"건너편 cell"의 bin도 검색해야 함) 필요 → 27→125 neighbor 검색 + 좌표
wrap 비용. 첫 컷 복잡도 회피로 PBC mode는 naive. 5k auto-disable이 perf
보호.

### J3. Same-lattice ref 가드 (renderer)

`primary.lattice === secondary.lattice` (object identity) 시에만 PBC
활성. 두 cell이 다르면 minimum-image 의미 모호 (어느 cell 기준?). 가장
안전한 선택: 다른 cell이면 raw cartesian.

Trajectory fixed-cell case (latticeMode='fixed'에서 모든 frame이 같은
lattice REF) 또는 사용자가 같은 cell의 두 변형을 비교하는 경우 PBC 작동.
NPT XDATCAR variable-cell이라도 frame i와 secondary가 다른 lattice면 raw.

### J4. cartToFrac 공식 inline

cifParser.ts에 같은 공식 있지만 import 시 cifParser → types → ... 의존
chain 끌고 옴. nnMatching는 currently dependency-free. inline 유지.

### J5. degeneracy 가드 — det < 1e-9이면 raw 반환

Singular lattice (zero volume) 시 invert 실패 → raw fallback. 정상 결정에서는
발생 안 함, 안전망 역할.

## Known limitations / deferred

1. **Spatial bin + PBC 결합** — 다음 컷에서 5k+ atoms PBC 케이스 perf 필요
   시 wraparound bin. v0.17.x.
2. **Lattice-mismatched 케이스** — relaxed vs unrelaxed cell parameter
   다를 때 PBC 활성화 안 됨. 사용자 선택으로 forcing PBC (어느 cell 기준
   선택) 옵션 추가 가능 — v0.17.x.
3. **체적 보존 변환** — relaxed/unrelaxed 비교 시 cell shape 자체가 다른데
   atom matching은 cartesian. fractional matching 옵션 (cell-relative)도
   고려 가능 — v0.18+.
