# v0.18.0 Feature 18.7 — Help overlay redesign + digits 1-4 shortcut

## Help card

- 720px wide V2 glass-3 card (`var(--v2-glass-3)`), `var(--v2-blur)`,
  `var(--v2-line-strong)` border, `var(--v2-r-lg)` corners,
  `var(--v2-sh-3)` shadow.
- Scrim picks up `backdrop-filter: blur(6px)` for the V2 dimming effect;
  `z-index: 200` (above the floating chrome's 13).
- Head: title at `--v2-t-title` (15px, weight 700) + small "press `?`
  anytime" sub. Close button uses the new `ICON.close` SVG.
- Body: **4-column grid** (Rotation / Navigation / Style / Mode) on
  ≥720px viewports; collapses to 2-column under 720px via media query.
- Rows use `.help-keys` (76px min-width key column) + label. `.kbd`
  V2-styled with surf-0 background, line-strong border, inset shadow,
  caps-size font.
- All Unicode key glyphs preserved (`←↓↑→ hjkl [] +-`) — no icon swaps
  here per user instruction "do not use icon or fonts of this".

## Digit shortcuts 1–4

New global `keydown` listener in `main.ts`:
- `1` → Ball & Stick, `2` → Space-filling, `3` → Stick, `4` → Wireframe
- Routes through `applyDisplayStyle(s, { dispatch: true })` so chip UI
  + saved-state writeback fire automatically.
- Skipped when an `INPUT`, `TEXTAREA`, or `[contenteditable]` is focused
  — digit entry in step / supercell inputs continues to work.
- Skipped when modifier keys (ctrl/meta/alt/shift) are held — keeps
  Ctrl+1..4 free for editor shortcuts.

## ICON.close added

Module-level `ICON` object in `crystalEditorProvider.ts` gains a `close`
entry (X path, same SVG_OPEN attributes). Used in the help-card close
button.

## Verify

- `npx tsc --noEmit` — clean
- `npm run build` — clean
- Manual: press `?` to open the help card; the four columns of shortcuts
  render in a glass card. Pressing `1`/`2`/`3`/`4` outside an input
  switches the display style and updates the chips.
