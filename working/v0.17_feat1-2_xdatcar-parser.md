## Autonomous judgments recorded

### J1. NPT header repeat heuristic

VASP NPT XDATCAR는 매 frame 앞에 7-line 헤더 (scale, 3 lattice, species,
counts)를 반복. 검출 휴리스틱:
- "Direct configuration=" 블록이 끝난 뒤 다음 비어있지 않은 라인이
  - 단일 숫자 토큰 (scale) AND
  - 다음 3 라인이 모두 3-숫자 vector → header repeat.
- 검출 시 lattice 업데이트 + 옵션 species/counts 라인 스킵 (VASP 5+ 형식).

VASP 4 형식 (species 라인 없음)은 첫 헤더에서 title로부터 추출 + NPT repeat에서
species 라인 부재 검출은 안 함 (VASP 4 NPT는 매우 드뭄).

### J2. Atom count drift 강한 throw

NPT에서 atom count가 frame별로 변하는 건 VASP 표준상 불가능. 그러나 corrupt
file에서 발생하면 silent error → frame 데이터 mismatch. 즉시 throw로
crystalEditorProvider error boundary가 토스트 띄움.

### J3. Cartesian configuration 지원

XDATCAR는 일반적으로 "Direct configuration=" (fractional)이지만 "Cartesian
configuration=" (cart) 도 가능. 두 케이스 모두 처리.

### J4. Single-frame entry는 첫 frame만

`parseXdatcar()`는 `parseXdatcarTraj()` 첫 frame 반환. CLI 등 single-frame
호출 사이트는 이 entry 사용. CLI 렌더러는 trajectory 미노출이라 첫 frame
시각화로 충분.

### J5. NVE/NPT fixture 동시 추가

J1의 NPT 검출 로직은 코드만 있으면 검증 안 됨. NVE fixture (단일 헤더 + 반복
configurations)와 NPT fixture (반복 헤더 + configurations)를 동시 추가하여
양 분기 모두 visual하게 검증. 이는 17.1.1 AXSF가 fixture로 안 검증한 NPT
대응.

   config = N M` 형식이거나 frame 간 atom 수 변동 가능 (VASP 표준 외). 표준만
   처리.

   frame만 렌더 가능 (parseXdatcar). v0.17.x trajectory CLI 진입.
