## Autonomous judgments recorded

### J1. Self-comparison (nacl→nacl) for sanity

먼저 동일 파일을 `--phase`로 지정해 NN matching이 자기 자신을 정확히
매칭하는지 확인. 결과: RMSD=0.000, matched 8/8, unmatched 0 — 기대치
정확. 만약 NN matching이 망가졌다면 self-self도 비매칭이 발생.

### J2. PBC-aware는 lattice 동일 시에만

`latticesEqual(a, b)` 9개 component element-wise 비교 — webview에서는
ref equality(`a === b`)로 충분했지만 CLI는 별도 parse 두 번이라 ref
equality 절대 성립 안 함. nacl-vs-silicon 시도에서 0/8 matched로 나오는
것 확인 (species 불일치) → PBC 분기 자체는 lattice 같을 때만 의미 있음을
재확인.

### J3. Perturbation fixture는 fractional 0.035–0.04 수준

5.64 Å cell × 0.04 = 0.226 Å — VESTA-style scale (1 Å ≈ 1.5 atom
radius)에서 visible하지만 inter-atom collision 안 일으키는 sweet spot.
3 atoms 만 perturb (Na2/Cl1/Cl3) → 5 zero-displacement + 3 non-zero =
회귀 catching 시 baseline 변화 명확. RMSD 예상치 √(2·0.226² + 0.198²)/√8
≈ 0.124, 실측 0.127 (PBC matched 8/8) → matching/계산 정합.

### J4. `--all-frames` 충돌은 v0.17.4 내부에서 exit 1

per-frame comparison (각 trajectory frame을 phase와 비교)은 의미 있는
기능이지만 stdout summary가 frame 수만큼 출력 + 사용자가 어떤 frame을
phase에 매칭하려는지 (모든 frame? 한 frame만? phase trajectory?) ambiguous.
v0.17.4에서는 fail-fast로 막고 v0.17.x에서 `--compare-trajectory` 별도
플래그로 다시 다루기로 deferred (working log에 기록).

### J5. stdout `[comparison]` 단일 라인 + grep-friendly prefix

`fmt = x => x.toFixed(3)` 일관 — log parsing 스크립트 작성 시 fixed
field width. CI 빌드 로그에서 `grep '\[comparison\]'`로 단일 라인 추출.

### J6. `__comparisonStats` 미존재 시 silent skip

`compareToPhase=true`인데 inline HTML이 phase가 빈 array라 비교 블록
실행 안 되는 경우 (이론적으로 pre-flight에서 잡히지만 방어적 코드).
`if (stats)` 가드로 stdout 출력 생략, 에러 없음.

## Known limitations / deferred

1. **Per-frame trajectory comparison** — `--compare-to-phase` +
   `--all-frames`는 v0.17.x에서 `--compare-trajectory` 별도 플래그로
   재설계 (어떤 frame을 phase로 쓸지 syntax 결정 필요).
2. **NN matching에서 multi-phase 지원** — 현재 첫 `--phase`만 사용. 후속
   phase는 시각 overlay만. 다중 phase pair-wise comparison은 별도 옵션
   (`--compare-pair=A,B`) 필요.
3. **JSON / CSV stats output** — stdout `[comparison]` 외에 `--stats-out
   file.json` 옵션은 v0.17.x.
4. **Per-pair displacement listing** — RMSD/max 외에 atom index별
   displacement table 출력 (DFT 정합도 디버깅용)은 v0.17.x.
