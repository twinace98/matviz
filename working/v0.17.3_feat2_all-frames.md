## Autonomous judgments recorded

### J1. Browser reuse via inner helper extraction

원래 plan §17.3.2 J7는 browser reuse를 v0.17.x로 deferred (단순성). 그러나
실구현 시 `render()` main loop에서 같은 browser/page를 N번 재사용하는 게
오히려 코드 단순 — `renderHtmlToPng(html, outputPath)` helper로 추출 +
loop 안에서 호출.

대안 (per-frame subprocess invocation): outer shell loop이 필요, 사용자
부담. 자율 결정으로 plan deferred 항목을 첫 컷에 포함 — 100 frame
trajectory가 ~3분 → ~1분으로 단축.

### J2. 1-indexed filename suffix (`_0001` not `_0000`)

내부 frame index는 0-indexed (renderer.setFrame, --frame N과 일관). 그러나
파일명 NNNN suffix는 1-indexed (`_0001`, `_0002`, ...) — 사용자 / ffmpeg
관례에 맞음. ffmpeg `-i md_%04d.png` 패턴이 1부터 시작 가정.

내부 vs 외부 인덱싱 트레이드오프 — 외부 (파일명) UX 우선. log 메시지에서
0-indexed 일관성은 `--frame N` 경로에만 한정.

### J3. Single-frame + `--all-frames` warn + 1-PNG 출력

대안: error로 reject. 그러나 사용자 워크플로우가 임의 파일을 자동 처리하는
스크립트일 때 single-frame .cif도 동일하게 처리되는 게 친화적. warn으로
"의도 확인" 시그널 + 1 PNG 출력으로 "안전한 default 동작" 보존.

### J4. `_NNNN` suffix는 4 → 5 digit 자동 확장 (>9999)

대안: 항상 5-digit (`_00001`, `_00002`). 그러나 일반 MD trajectory는 1k–10k
frame이라 4-digit 익숙. 9999 초과 시만 자동 확장 → 일반 케이스 가독성 유지
+ edge case 안전.

ffmpeg `-i md_%04d.png` (4-digit)와 `-i md_%05d.png` (5-digit)는 호환 안
됨 — 사용자가 9999 초과 trajectory에서는 ffmpeg 패턴도 5-digit으로 조정
필요. SKILL.md 노트는 4-digit case만 (일반).

### J5. Conflict warn over silent precedence

`--all-frames` + `--frame N` 동시 → all-frames 우선 + warn. silent
precedence는 사용자 의도 모호 (왜 N이 무시됐는지 모름). warn으로 명시.

### J6. Volumetric은 trajectory 첫 frame에서 추출

trajectory + isosurface 결합은 17 범위 외. 그러나 첫 frame의 volumetric을
`--all-frames` 모든 frame에 inherit (loop 안에서 같은 volumetricJSON 사용).
사용자가 시각적으로 atom 재생만 원하는 경우 합리적 default.

per-frame 변동 volumetric은 v0.17.x extended XYZ properties pipeline 후.

### J7. Per-frame stdout "Saved" 로그

100 frame이면 100줄 log. 노이즈가 클 수 있지만 progress visibility 가치.
`--quiet` 옵션은 v0.17.x 후보. ffmpeg 자체도 frame-level log 보여주므로
사용자 익숙.
