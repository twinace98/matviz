# v0.15 feat 1 — Sphere impostor

## 신규 파일
- `src/webview/sphereImpostor.ts` — `SphereImpostorMesh` (InstancedMesh subclass) + `createImpostorMaterial()` (GLSL 3.00 ShaderMaterial) + 공유 quad geometry.
  - Vertex: instance matrix에서 center + uniform-scale radius 추출, view-space에서 camera-aligned quad 배치.
  - Fragment: view-space ray-sphere intersection → discard | gl_FragDepth + Phong-lite. 직교/원근 분기 (`uOrtho`).
  - Custom `raycast()` — 각 instance 당 ray-sphere 테스트. 기존 `onCanvasClick` / `updateSelectionHighlight` 경로 그대로 동작.

## renderer 변경
- `impostorMode: 'auto' | 'on' | 'off'` 필드 + 게터/세터. `auto` 시 N ≥ 2000 자동 활성.
- `buildAtoms()` 분기: impostor 경로에서는 quad geometry + `SphereImpostorMesh` 사용, 기존 경로는 `SphereGeometry`.
- `setCameraMode()`에서 `uOrtho` 업데이트, `renderFrame()`에서 world→view light direction 변환하여 `uLightDir` 갱신 (Phong atom/bond 과 동일한 광원).
- `PersistedState`에 `impostorMode` 추가.

## 셰이더 초기 버그 (수정됨)

Three.js 0.170 `ShaderMaterial` + `glslVersion: GLSL3` prefix 주입 규칙:

- **Vertex prefix**: `modelMatrix`, `modelViewMatrix`, `projectionMatrix`, `viewMatrix`, `normalMatrix`, `cameraPosition`, `isOrthographic`, + `USE_INSTANCING` 시 `in mat4 instanceMatrix`, + `USE_INSTANCING_COLOR` 시 `in vec3 instanceColor`, 언제나 `in vec3 position; in vec3 normal; in vec2 uv;` 자동.
- **Fragment prefix**: `viewMatrix`, `cameraPosition`, `isOrthographic` 뿐. `projectionMatrix` 등은 **자동 선언 안 됨**.

초기 구현은 VERT 에 `in vec3 position; in mat4 instanceMatrix; in vec3 instanceColor;` 를 중복 선언 → duplicate identifier 컴파일 에러. 또한 FRAG 에서 `projectionMatrix` 을 선언 없이 사용 → undeclared identifier.

Puppeteer + SwiftShader 로 WebGL2 에러 로그 캡처하여 확인. 둘 다 `sphereImpostor.ts` 에서 고정:
- VERT: 중복 `in` 선언 제거 (Three.js auto-inject 에 맡김).
- FRAG: `uniform mat4 projectionMatrix;` 직접 선언 (Three.js 는 program-level 에서 값 바인딩 해주므로 선언만 하면 동작).
