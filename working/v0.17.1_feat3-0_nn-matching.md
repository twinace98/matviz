## Autonomous judgments recorded

### J1. Greedy NN over Hungarian optimal assignment

Greedy: O(N) primary atoms × O(K) candidates per atom (K ≤ 27 with binning).
Hungarian: O(N³). 시각화 use case에서 두 알고리즘 차이는 borderline 케이스
(여러 atom이 같은 secondary atom을 두고 경쟁) 한정. 전형적 비교 (relaxed
vs unrelaxed, 같은 cell의 작은 displacement)에서는 결과 동일.

90%+ 케이스에 충분. v0.18+ 정밀 RMSD 분석 필요 시 Hungarian 도입 고려.

### J2. Same-species 한정 + species-grouping

매칭은 같은 species 내에서만 — Cl이 Na로 대응되는 매칭은 의미 없음. 코드
구현은 secondary를 species별로 그룹화 → primary atom의 species에 해당하는
candidates만 검색. 자연스럽게 N²이 ΣNᵢ²로 줄어듦.

### J3. Spatial bin for N>100

Threshold 100은 양 케이스 (small naive, large bin) 통과 점. 더 정확한 임계는
benchmark 필요하지만 현 시점에서 합리적. bin size = max(threshold, 1.0) →
27 bin 검색으로 threshold 반경 sphere 안 atom 모두 cover.

### J4. usedSecondary set으로 one-to-one 강제

같은 secondary atom이 두 primary atom에 동시 매칭되는 것 방지. greedy +
insertion order 따라 결정성 보장. 매칭 결과는 입력 순서에 의존하지만 시각
적으로는 무관.

### J5. Threshold 위반 시 unmatched 분리

거리가 threshold를 초과하면 unmatched 배열에 primary 인덱스 추가. variant
비교에서 atom 추가/제거 케이스 처리. 17.3.1 렌더러는 unmatched는 화살표
미생성.

### J6. Test 1-3 (identity/drift/swap) — 알고리즘 정확성 / Test 4 — bin 경로

4 케이스로 small + large 둘 다 cover. Test 4는 deterministic LCG seed 사용
(reproducible). bin 활성화는 N=200 (>100) 자동.

## Known limitations / deferred

1. **Hungarian optimal assignment 미사용** (J1) — borderline conflict 시 sub-
   optimal. RMSD 분석 워크플로우 필요 시 v0.18+.
2. **PBC 무시** — atom이 periodic boundary 넘어 이동 시 (XDATCAR NPT에서) 큰
   displacement 화살표 생성. v0.17.x: optional `--periodic` flag로 minimum-
   image distance 사용.
3. **Bipartite cross-species 미지원** — 정의상 안 함. Same-species만.
