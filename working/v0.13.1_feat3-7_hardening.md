# v0.13.1 · Features 3–7 hardening

**Status:** complete
**Date:** 2026-04-17

## 13.1.3 · Ambiguous-extension priority + "Open in MatViz" button
- `package.json`: split `customEditors` into two entries. Unambiguous formats stay `priority: "default"`. Ambiguous `*.in / *.out / *.stdin / *.stdout / *.pw` moved to a second entry with `priority: "option"` so VSCode opens them as text by default.
- Added `editor/title` menu contribution — "Open in MatViz" button (icon `$(graph)`) visible only when `resourceExtname =~ /^\\.(in|out|stdin|stdout|pw)$/`.
- `src/extension.ts`: registered `matviz.openInViewer` — resolves the active editor's URI and runs `vscode.openWith(uri, 'matviz.crystalViewer')`.
- `src/parsers/index.ts`: `.stdin` / `.stdout` now route to `parseQE`; `.in` routes to aims if the aims signature matches, else QE.

## 13.1.4 · QE parser silent-default removal
- `src/parsers/qeParser.ts`: throw when no `CELL_PARAMETERS` / `ATOMIC_POSITIONS` block was consumed. Previously returned a placeholder 10×10×10 cell with no atoms — hid real parse failures behind an apparently-valid structure. Error boundary from 13.1.2 surfaces the thrown message.

## 13.1.5 · Renderer inline-material registry
- `src/webview/renderer.ts`: added `private materials: THREE.Material[]` + `trackMat<T>(m: T): T` helper.
- Wrapped 11 inline allocations: lattice plane Phong, two isosurface Phongs, axis-arrow shaft/head basic pair, axis sprite, origin basic, measurement dashed line, measurement sprite, bond `LineSegments` basic (vertex-colors), polyhedra Phong + edges basic, label sprite, outer cell-box basic, inner supercell dashed.
- `disposeAllMaterials()` now iterates the registry in addition to the `materialCache` map.

## 13.1.6 · CLI renderer `try/finally`
- `scripts/render.ts`: entire `page.*` lifecycle wrapped in `try { … } finally { unlink tmp; browser.close() }` so Puppeteer can't leak a child process on parser/runtime errors. `process.exitCode` replaces the mid-flow `process.exit(1)` calls inside the try block.

## 13.1.7 · CSP `'unsafe-inline'` removal
- `src/editor/crystalEditorProvider.ts`: dropped `'unsafe-inline'` from the `style-src` directive.
- HTML template converted inline `style="display:none"` / `style="width:100%"` attributes into classes.
- `media/styles.css`: added `.hidden { display: none }` and `.sc-input.full-width { width: 100% }`.
- `src/webview/main.ts`: `initTogglePanel` rewritten to toggle the `hidden` class (inline-style check was broken against CSS-driven default). `iso-section` reveal removes the class explicitly before clearing inline style.
