# v0.18.0 Feature 1 — Groundwork (unified SVG icons + step-angle/zoom stepper)

**Status:** code complete, both quality gates green (build + tsc), pre-commit.

## What

Carries forward the unified inline-SVG icon set and the custom numeric stepper
work that landed visually in this session, scoped down to just the bits V2
keeps:

- 24 inline-SVG icons (16×16, 1.5px stroke, currentColor, no fill, round
  caps) replace the Unicode/emoji glyphs in `crystalEditorProvider.ts`'s
  top toolbar and mode bar. Defined as raw strings on a module-level
  `ICON` object; spliced into the HTML template literal.
- Palette toggle (`☾ ↔ ☀`) swaps between `SVG_MOON` and `SVG_SUN` defined in
  `src/webview/main.ts`. Two strings duplicated across the bundle boundary
  (extension host vs. webview) since they can't share a TS module.
- `step-angle` and `step-zoom` inputs swapped from `type="number"` to
  `type="text" inputmode="numeric"`, wrapped in a `.num-wrap` box with
  stacked `.num-step.up` / `.num-step.dn` buttons. New
  `setupNumberStepper()` helper in `main.ts` binds button click + Up/Down
  keys, dispatches synthetic `change`/`input` events, clamps via wrapper
  `data-min`/`data-max`, and tolerates transient empty/invalid mid-edit
  states (clamp on blur).

## What was rolled back from this session's V1 work

- **Supercell inputs (`sc-a/b/c`)** reverted to plain `<input type="number">`
  — V2's Feature 18.6 will replace them with the `− N +` horizontal stepper
  layout (no upper bound), which is structurally different from the generic
  `.num-wrap`. Re-using the V1 stepper would mean dismantling it again
  immediately, so the sc-* inputs sit untouched until 18.6.

## Files

- `src/editor/crystalEditorProvider.ts` — ICON object, ~24 button glyph
  swaps, step-angle/zoom inputs wrapped, sc-a/b/c restored
- `media/styles.css` — `.num-wrap`, `.num-steps`, `.num-step` rules; SVG
  `display: block` on `.bar-btn svg, .mode-btn svg`
- `src/webview/main.ts` — `SVG_MOON`/`SVG_SUN`, palette innerHTML swap,
  `setupNumberStepper()`, registered for step-angle/step-zoom only

## Verify

- `npx tsc --noEmit` — clean
- `npm run build` — clean
- Manual: open any fixture; toolbar buttons render as line icons; ▲/▼
  beside Step(°) / Step(%) clamp 1–90 / 1–50; ArrowUp/Down keys also work.

## Why a separate commit

Folds in groundwork that was built before the v0.18 plan was approved, so
the version's history starts from a clean visual-only baseline before the
glass-tokenization work in 18.2 lands.
