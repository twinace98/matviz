# v0.18.0 Feature 18.6 — Panel content surfaces (chips + iOS switches + supercell + info pill + axis gizmo)

Five bundled subsurface changes — all sit inside (or relate to) the side
panel. Bundled per the impl spec because they share the panel-restructure
context introduced in 18.5 and several touch the same JS init code.

## 6a — Style chips

- HTML: `<select id="display-style">` replaced with
  `<div id="display-style-chips" class="chips">` of 4 `.chip` buttons
  (`data-style="ball-and-stick"` etc.). Active chip gets `--v2-surf-accent`
  + `--v2-line-focus` border.
- JS: `applyDisplayStyle(s)` is now the single entry point — toggles the
  active chip and calls `renderer.setDisplayStyle(s)`. Click delegation on
  the chip container; saved-state restore routes through the same.
- `styleSelect` const retained as alias of `styleChips` (HTMLElement) so
  the `addEventListener('change', debouncedSave)` pattern at the bottom of
  the file keeps working — chip clicks dispatch `change` synthetically.
- Tooltips on each chip mention the digit shortcut `(1)` … `(4)` even
  though the actual binding lands in Feature 18.7.

## 6b — iOS toggle switches

- New `setupToggleSwitches(root)` in `main.ts` walks every `.toggle
  input[type=checkbox]` once at startup and injects a sibling `.switch
  > .switch-thumb` span. Native checkbox is visually hidden via CSS
  (`position:absolute; opacity:0; pointer-events:none`) but stays in the
  label for click-toggle, focus, and `change` events.
- Visual: 36×20 pill, VSCode focus blue when on, 16×16 white thumb that
  slides 16px on transition (200ms cubic-bezier).
- Idempotent — `dataset.switchInjected` guards re-runs. Per-element atom
  visibility checkboxes (`.prop-vis`) intentionally NOT switched (different
  class, smaller form factor, kept native for grid density).

## 6c — V2 supercell stepper (`− N +`, no upper bound)

- HTML: each axis is `<div class="sc"><div class="sc-l">a</div><div
  class="sc-steppers"><button>−</button><input type="text"
  inputmode="numeric"><button>+</button></div></div>`.
- Upper bound **removed**. `data-axis-min="1"` per axis. (The 5000-atom
  bond-detection skip is the real safety net, surfaced via the existing
  bond-skip hint banner.)
- JS: `setupSupercellStepper()` binds buttons + ArrowUp/Down keys, dispatches
  synthetic `change` so `updateSupercell()` fires.
- Caveat for documentation: persisted `supercell` arrays from older
  sessions can now be values >5 in subsequent sessions.

## 6d — Info pill (canonical formula readout)

- New `<div id="info-pill"><span id="pill-formula"></span><span
  id="pill-meta"></span></div>` after `#side-panel`.
- Always visible at `bottom: 16px; left: rail+12`; shifts to
  `left: rail+12 + side-panel-w + 12` when `body.panel-open` is set.
- Populated by `setInfoPill(formula, metaHtml)` from the `loadStructure`
  and `loadTrajectory` message branches in `main.ts`. The previous
  `#info` element inside the panel is gone — pill is canonical.
- Meta uses ` · ` separators with `<b>` around the numeric values
  (atom count, volume); inline-styled fallback if `getStructureInfo()`
  returns null.

## 6e — Axis gizmo bottom-right

- `axisIndicator.ts` `render()` swaps from hard-coded `(4, 4, size, size)`
  bottom-left viewport to a right-aligned position computed from
  `renderer.domElement.clientWidth - margin - size`. Margin = 16 to match
  V2's `bottom: 16; right: 16` spec.
- Pure WebGL viewport relocation; no DOM glass-card decoration. The
  decoration would have meant alpha-overlaying the canvas pixels (which
  would tint the axis-arrow rendering), so kept indicator on canvas as-is.

## Panel section polish

- `.panel-section` gets `border-top: 1px solid var(--v2-line)` + 12px
  vertical padding (was 4px). First section drops the border. V2-style
  vertical rhythm.
- `.panel-label` swapped to V2 caps style: 10px, weight 600, letter-spacing
  1.1px, color `--v2-fg-3`.
