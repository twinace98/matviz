## Autonomous judgments recorded

### J1. 16.3만 우선 포팅 (16.1/16.2/16.4 보류)

사용자 명시 요청 ("가장 많이 쓸 기능"). DFT magnetic structure 보고서 워크플로우
에서 가장 빈번:
- VASP NiO/CoO/Fe/Cr 계산 → POSCAR title MAGMOM → CLI render → 보고서.
- ellipsoid (16.1)는 X-ray crystallography 보고서에 더 적합 (덜 자주).
- partial occupancy (16.2)는 광물학에서 사용 (덜 자주).
- Wulff (16.4)는 표면 에너지 계산 워크플로우 (less common).

→ 우선순위에 따라 16.3만 포팅. 나머지는 사용자 요청 시 같은 패턴으로 추가.

### J2. Webview 모듈을 import 불가 — inline 재구현

`scripts/render.ts`는 Puppeteer self-contained HTML을 string template으로
생성. `import { MagneticArrowRenderer }`는 string 내부에서 안 됨.

대안:
- A. Inline 재구현 (현재 선택). webview 모듈과 두 곳 유지 부담.
- B. `dist/render-helpers.js`에 추가 export. webview는 그것을 다시 import.
  유지 단일화는 가능하지만 helpers는 ESM browser bundle이고 webview는 IIFE
  → import 호환성 추가 작업.
- C. CLI inline HTML 자체를 모듈 시스템화. 큰 리팩토링.

**선택: A.** 현 v0.16.3 fix (`0c23a2c`)에서 발견한 `vertexColors:true` 함정도
inline 코드에 명시 주석으로 박아 동일 실수 재발 방지. 두-곳 유지 부담은
PostToolUse hook으로 SKILL.md/README.md 동기화 reminder 띄우는 패턴과 같이
관리.

### J3. Visual harness fixture 신규 추가

CLI 16.3이 동작하는지 자동 catching 위해 5번째 fixture 추가:
- `magmom-arrows/scene.json` — view=a (arrow가 카메라에 수직 → 잘 보임), redblue.
- baseline 생성 → 향후 회귀 발생 시 ΔRGB > 게이트 → catch.

대안: viridis colormap도 별도 fixture. 보류 (단일 fixture로 instance render +
shaft/tip/색상 path 모두 회귀 catching).

### J4. View 'a' 선택 (시각 검증 명료성)

`view=c`로 첫 시도 → arrow가 z축 방향이라 카메라 축과 평행 → 거의 안 보임
(점으로 보임). `view=a`로 변경 → arrow가 화면 평면 내 → 명료한 ↑↓ 표시.

→ fixture는 `view=a`로 고정. 사용자가 다른 view로 렌더할 때는 magnetic
direction과 view direction 관계 고려해야 함. SKILL.md recipe 예제에서도
`--view a` 사용.

### J5. Hook reminder 4번 누적 처리

`scripts/render.ts` 편집마다 PostToolUse hook이 SKILL.md/README.md 동기화
reminder 발생 (4회). 모두 commit 전 일괄 처리.

→ 이게 의도된 워크플로우 (CLAUDE.md `.claude/settings.json` hook 정의). 자율
진행 중에도 hook context를 인식하고 마지막에 일괄 동기화 → CLAUDE.md
"각 commit에 SKILL.md 동기화" 규칙 준수.

## Known limitations / deferred

1. **INCAR auto-discovery** — 여전히 미지원. POSCAR title MAGMOM이거나 CIF
   moment loop만 동작. v0.16.x.
2. **압축 MAGMOM (`4*1.0`)** — webview 동작과 동일하게 reject + console warn.
3. **16.1 ellipsoid CLI** — symEigen 수학 + per-instance 4×4 매트릭스 inline
   포팅 (~150 LOC). 사용자 요청 시 추가.
4. **16.2 partial occupancy CLI** — per-atom transparent Mesh inline (~50 LOC).
   사용자 요청 시 추가.
5. **16.4 Wulff CLI** — `wulff.ts` inline + ConvexGeometry (이미 helpers에
   export 있음). 사용자 요청 시 추가.
6. **`matviz-render` skill 재설치 필요** — `npm run install-skill` 또는
   `npm run install-all`로 SKILL.md 사본을 `~/.claude/skills/`에 갱신.
