## Autonomous judgments recorded

### J1. INCAR 자동 검색 deferred

Plan §16.3은 POSCAR 인접 INCAR 검색 명시. 그러나 webview는 file system 접근
불가 → extension host 측 (Node.js, `fs.readFileSync`) 작업 필요.

대안 비교:
- A. 이번 commit에 extension.ts 변경 추가 (POSCAR 파일 핸들러 후크에서 인접
  INCAR 검색).
- B. **Deferred (16.x). POSCAR 코멘트 라인 MAGMOM만 일단 지원.**

선택 B 근거:
- Webview-only 변경으로 sub-commit 깔끔 유지.
- POSCAR comment에 MAGMOM 박는 패턴은 흔함 (사용자가 직접 박거나 ASE/pymatgen
  export 시).
- INCAR 검색은 file system + 디렉토리 권한 + UNC path 처리 등 잡일 추가.
- 16.x로 분리해도 워크플로우 단절 없음 (사용자가 POSCAR 코멘트 추가 가능).

기록: working log + commit Known limitations 섹션.

### J2. Arrow geometry 분할 — shaft + tip 별도 InstancedMesh

대안:
- A. Shaft + tip을 한 BufferGeometry로 합쳐서 InstancedMesh 1개. 작업 복잡
  (vertices merge + UV 정렬).
- B. **별도 InstancedMesh 2개 (shaft, tip). ✅** 자명한 매트릭스 변환 (atom
  중심 + 길이/2 vs atom 중심 + 길이). draw call 1+1.

근거: 단순 + draw call 2개는 무시 가능 비용. 모든 arrow가 같은 shaft geometry
와 tip geometry 공유.

### J3. Per-instance color via `instanceColor` + `vertexColors: true`

Three.js InstancedMesh는 `instanceColor`로 per-instance RGB 가능 (alpha 없음).
shaft/tip 둘 다 `MeshPhongMaterial` + `vertexColors: true` → 각 instance가
colormap 색상으로 tinted. 단일 material per shape.

### J4. Colormap 디스패치 — Red-Blue diverging vs Viridis

기본값 Red-Blue diverging (collinear AFM/FM 시스템에서 가장 직관적: ↑=빨강,
↓=파랑). Viridis는 sequential perceptually-uniform 옵션 (4-stop 근사로
하드코드, 전체 colormap 라이브러리 도입 회피).

UI에서 radio 선택 가능. v0.16.3 기준 두 가지만 — 추가 colormap은 16.x.

### J5. Magnetic arrow는 atom 분기와 독립 (ellipsoid/partial과 다름)

Ellipsoid는 atom을 *대체*하므로 buildAtoms 안에서 split. partial occupancy도
atom *변형*. magnetic arrow는 atom 위에 *오버레이*하므로 별도 group + 별도
경로. atom 분기 영향 없음 → ellipsoid + partial occupancy + magmom 셋 다 동시
on 가능.

코드 위치: `renderer.ts buildVisuals()`에 별도 블록.

### J6. Symmetry-op rotation 미적용 (U_aniso 한계 carry-over)

CIF moment의 symmetry expansion 시 R·m 변환 미적용. 같은 사이트의 모든
symmetry-equivalent atom이 같은 moment vector를 inherit. AFM 사이트에서는
오류 (예: 2 자리에서 +m, 다른 자리에서 −m 이어야 함).

POSCAR MAGMOM은 모든 atom (full list)이라 영향 없음. CIF는 symmetry 미적용 시
정확 (P1 또는 explicit listing).

기록: working log + commit + cifParser.ts 주석.

### J7. Zero-moment atom 스킵 (no arrow drawn)

`length(m) < 1e-4` → arrow 미생성. InstancedMesh slot 낭비 회피 + 시각적으로도
zero moment 표시 의미 없음.
