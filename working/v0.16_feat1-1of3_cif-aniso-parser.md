## Autonomous judgments recorded

### J1. Pre-existing loop-tracking bug discovered

기존 `cifParser.ts` (lines 56–69)는 단일 `loopColumns`/`loopRows` 배열만
유지하고 새 `loop_` 시작 시 이전 데이터를 wipe함. 정확히는:

```
if (line === 'loop_') {
  if (inAtomLoop && loopRows.length > 0) {
    inLoop = false;
    inAtomLoop = false;          // ← inAtomLoop 강제 false
  }
  ...
  if (!inAtomLoop || loopRows.length === 0) {  // ← !inAtomLoop는 항상 true
    loopColumns.length = 0;       // ← 결과: 항상 wipe
    loopRows.length = 0;
  }
}
```

선재 fixture 모두 atom loop가 마지막이라 발현 안 했지만, aniso CIF는 atom
loop 뒤에 aniso loop가 와서 깨짐. 16.1 작업 중 발견 → **same commit에서 수정
**(부수 fix). 새 구조: `parsedLoops: CifLoop[]` 누적 후 column 이름으로 매칭.

근거 파일: 새 `cifParser.ts` lines 27–117 (scanner) + 138–145 (atom/aniso 매칭).

### J2. Symmetry-expansion 시 Uᵢⱼ 회전 미적용

CIF `_atom_site_aniso_*`는 asymmetric unit에 대해 정의됨. symmetry op `R`로
좌표 `p → R·p` 회전 시 U tensor도 `U → R·U·Rᵀ`로 회전해야 *기하학적으로 정확*
함. 그러나 현재 구현은 parent label에서 U를 그대로 inherit (`{ ...u }`).

**선택 근거:**
- diagonal-dominant U (대부분의 일반 광물)는 회전이 거의 무차이 (대각 성분만 의미).
- off-diagonal U는 시각적으로 mis-orient되지만, 16.1 sub-commit (1/3)의 핵심
  목표는 *데이터 플로우 contract* 수립이지 정확한 결정학 렌더가 아님.
- 회전을 도입하려면 (a) symop string에서 R 행렬 추출 (이미 `parseSymOp`로
  가능), (b) 각 expanded atom이 어떤 symop으로 생성됐는지 추적 (현재 미구현),
  (c) `R · U · Rᵀ` 적용 — 추가 ~30 LOC.

**Deferred to:** 16.1 sub-commit (3/3) 또는 16.x. 코드에 `TODO 16.x` 주석으로
표시됨 (`cifParser.ts` lines 18–23 + 220–223).

   (희귀)에서 ellipsoid 방향 부정확. 16.1 (3/3) 또는 16.x.

3. **CIF aniso의 다른 형식 미지원** — `_atom_site_aniso_beta_*` (β-form;
   드물지만 일부 구형 CIF), `_aniso_oc_*` (occupancy-modulated) 등. 추가 시
   같은 anisoLoop dispatch에 케이스 추가만 필요.
4. **Aniso 단독 (atom 미존재) CIF**: 비정상 입력. 현재 파서는 atomLoop 미발견
   시 빈 species/positions 반환. 토스트는 v0.13.1 boundary가 처리.
5. **POSCAR / XYZ / PDB의 thermalAniso 미지원** — 이 형식들은 anisotropic
   displacement 표현 자체가 없음. CIF 단독 진입.
6. **Symmetry-expansion path test 부재** — sub-commit (1/3) 기준 합성 fixture
   는 P1. 실제 symmetry expansion + aniso 통합 경로는 (3/3)의 calcite fixture
